# HyLink 技术导航站

HyLink 是一个面向开发者与技术爱好者的轻量级开源导航与资源聚合平台，旨在解决个人或团队在浏览、检索、整理日常高频使用的技术文档、工具站点、影音素材库及开发资源时面临的链接分散、管理效率低下的问题。该项目并非一个传统的 Web 应用框架，而是一套基于纯静态 Markdown 与结构化 YAML 数据构建的快速索引系统，配套提供可部署的 Node.js 服务端渲染方案，用于生成高度可定制的导航门户。

项目目标用户包括前端工程师、运维人员、技术团队管理者以及需要频繁切换多个外部服务或内部系统的 IT 从业者。HyLink 通过提供统一的数据录入规范与极简的渲染层，允许用户以编写文本配置的方式维护自己的“技术外链宇宙”，并支持一键导出为 HTML 静态站点或集成至现有内部知识库中。

## 功能概览

- 多级分类索引体系：支持无限层级的目录结构，允许用户按技术栈、业务域、使用频率等维度自由组织链接资源，分类数据与链接实体解耦，便于复用与迁移。

- 外链健康状态探测：内置基于 HTTP 状态码与响应超时的被动探测模块，可定期检查所有收录链接的可达性，并在管理面板中标记异常链接，降低维护成本。

- 全文与标签检索：为每个链接条目自动生成关键词索引，支持标题、描述、标签组合检索，检索逻辑基于轻量级倒排索引实现，无需外部搜索引擎依赖。

- 数据导入与导出：支持从 CSV、JSON Lines 格式批量导入已有链接数据，同时支持导出为纯 Markdown 表格、JSON 结构化数据或符合 Hugo/VuePress 格式的配置文件，便于与其他静态站点生成器联动。

- 视图模板自定义：渲染层采用 Nunjucks 模板引擎，用户可修改分类列表、卡片样式、详情页布局等所有视图组件，无需修改核心数据解析逻辑。

- 响应式移动端适配：默认 UI 基于 Flexbox 与 CSS Grid 构建，在桌面、平板与手机屏幕下均能保持良好的阅读与点击体验，导航按钮与搜索栏针对触屏操作进行优化。

- 本地离线缓存策略：通过 Service Worker 预缓存分类元数据与公共样式资源，在无网络环境下仍可浏览已打开过的分类页面，提升内网或弱网场景下的可用性。

## 应用场景

技术团队内部知识库入口管理：开发团队通常拥有多个内部系统（如 GitLab、Jenkins、SonarQube、Confluence）以及外部依赖（如 Docker Hub、Maven 中央仓库、NPM 镜像站）。HyLink 可作为团队统一的导航入口，由技术负责人维护一份公共分类配置文件，所有成员通过内网访问同一站点，避免每次都手动输入地址或翻找浏览器书签。

个人开发者的工作台聚合：独立开发者或远程工作者可将每日必开的工具站点（API 文档、设计资源、图标库、颜色工具、字体库）归入“日常工具”分类，将正在跟进的开源项目仓库、Issue 列表、PR 页面归入“项目跟踪”分类，配合浏览器起始页插件使用，显著减少标签页切换时间。

技术培训与教学辅助：在培训机构或高校实验课中，讲师可以将课程所需的在线编译器、代码示例仓库、习题提交入口、在线评测系统等链接预先整理至 HyLink 中，学生通过一个地址即可访问全部教学资源，避免因输入错误导致的课堂时间浪费。

影音素材与媒体资源快速索引：对于涉及多媒体内容创作或素材收集的技术人员（如游戏开发、UI 动画设计），可利用 HyLink 管理多个视频素材站、音效库、无损图片站点及字幕资源站，例如项目资源列表中收录的影音类站点，均可通过分类标签进行分组，实现素材来源的统一调度。

## 快速开始

以下命令演示如何从 GitHub 克隆项目仓库、安装依赖并启动开发服务器。默认使用 Node.js 18 LTS 及以上版本。

```bash
git clone https://github.com/hylink-org/hylink-navigator.git
cd hylink-navigator
npm install
npm run dev
```

执行完上述步骤后，打开浏览器访问 http://localhost:3000 即可预览默认示例导航数据。如需修改或扩充链接资源，请编辑 `data/links.yml` 文件并按照已有格式追加条目，保存后开发服务器会自动热重载。

## 贡献指南

我们欢迎各类形式的贡献，包括但不限于新增功能、修复缺陷、改进文档或优化性能。请遵循以下步骤提交您的更改：

1. 查阅问题追踪列表：访问 GitHub Issues 页面，查找未被认领的 “help wanted” 或 “good first issue” 标签问题。若您有新功能想法，请先创建一个 Issue 与维护者讨论其合理性与实现范围，避免无效劳动。

2. 派生仓库并创建特性分支：将本仓库 Fork 至您的账户下，然后基于 `main` 分支新建一个描述性的分支名称，例如 `feature/add-dark-theme` 或 `fix/parser-encoding-issue`。请确保分支名称简洁且反映变更内容。

3. 编写代码并添加测试：所有新功能或缺陷修复必须包含对应的单元测试或集成测试，测试文件放置于 `tests/` 目录下，且保持与源码目录结构对应。同时更新 `docs/` 下相关文档，说明改动影响的范围及使用方法。

4. 提交前运行代码检查与测试套件：执行 `npm run lint` 检查代码风格，执行 `npm test` 确保全部测试用例通过。若修改了数据解析逻辑，请额外使用 `data/links.yml` 示例文件进行本地构建验证 (`npm run build`)。

5. 发起 Pull Request 至主仓库：在 PR 描述中清晰列出变更列表、关联的 Issue 编号以及测试覆盖情况。维护者将在 3 个工作日内进行审查，可能需要您根据反馈进行补充修改。合并后您的贡献将出现在下一版本发布记录中。

## 常见问题

问：如何更新已收录链接的图标或缩略图？

答：HyLink 默认不存储任何图片资源，所有图标均通过链接条目中的 `icon_url` 字段引用外部地址。您可以直接修改 `data/links.yml` 中对应条目的 `icon_url` 值为新的图片链接，保存后重新构建或重启开发服务器即可生效。若未设置该字段，系统将使用分类首字母生成文本占位图标。

问：生产环境部署后，链接健康状态探测功能为何不工作？

答：链接健康状态探测在开发环境下默认启用，但在生产环境中出于性能考虑默认关闭。如需开启，请在 `config/ecosystem.config.js` 或环境变量中将 `HEALTH_CHECK_ENABLED` 设置为 `true`，并配置 `HEALTH_CHECK_INTERVAL`（单位分钟）。同时请确保部署环境可以访问外网，否则探测请求可能因网络策略超时失败。

问：能否将 HyLink 与现有的 VuePress 或 Docusaurus 项目集成？

答：可以。HyLink 提供了数据导出命令 `npm run export:markdown`，该命令会将所有分类与链接条目输出为一个或多个 Markdown 表格文件，您可以直接将这些文件放入 VuePress 或 Docusaurus 的文档目录中引用。另外，也支持导出为 JSON 格式 (`npm run export:json`)，供其他前端框架通过 API 或静态数据方式加载。
