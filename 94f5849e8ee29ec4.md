# PaperBridge

PaperBridge 是一个面向科研工作者与工程技术人员的学术资源导航与镜像聚合平台。项目定位于对分散于网络各处的开放获取论文站点、预印本仓库及学术镜像站进行系统化梳理与索引，通过统一的目录结构与快速启动脚本，帮助用户在受限网络环境下高效获取外文文献与学术资源。其核心目标用户包括高校研究生、科研机构工程师、自由研究者及信息科学领域的开发者。

PaperBridge 本身不存储任何侵权内容，亦不提供文件下载服务，仅作为公开可访问链接的整理与分类参考。项目所有资源链接均来源于社区公开分享，旨在降低信息检索的时间成本，提升学术工作的连续性。项目采用静态 Markdown 索引结构，配合轻量级本地服务器脚本，可在数秒内完成私有化部署，适用于个人工作站、实验室内部网络及小型研究团队的知识管理场景。

## 功能概览

- **按学科分类的镜像站索引**：将收集到的资源链接按计算机、物理、数学、综合等学科门类进行标签化整理，便于快速定位。

- **站点存活状态自动检测**：集成基于 HTTP 状态码的简易可用性检查脚本，可对列表中的 URL 进行批量连通性测试，输出不可达站点报告。

- **自定义资源分组与备注**：支持用户在本地配置文件中为每个链接添加分组标签、来源说明及维护备注，所有修改保留于本地，不影响远程同步。

- **内置轻量级静态服务器**：提供基于 Python HTTP 服务器的一键启动命令，可在局域网内快速分享索引页面，适用于无图形界面的服务器环境。

- **Markdown 表格化文档呈现**：所有资源列表以标准化 Markdown 表格呈现，包含站点名称、URL、可用状态、所属类别及最后检查时间，便于直接复制或转换为其他格式。

- **定时更新提醒机制**：内置简单的日期比较逻辑，当本地索引文件最后修改时间超过 30 天时，在启动时输出提示信息，建议用户手动复核链接有效性。

- **多终端适配的响应式页面模板**：附带基础 CSS 样式，确保在手机、平板及桌面浏览器下均可获得良好的阅读体验，无需额外安装阅读器。

## 应用场景

1. **实验室内部文献导航站**：研究团队可将 PaperBridge 部署于实验室共享服务器，所有成员通过浏览器访问同一份索引页面，快速跳转至常用预印本站点，避免重复收藏书签。适用于计算物理、材料模拟等需要频繁查阅 arXiv 及专业镜像的领域。

2. **个人学术工作台配置**：研究生或博士后可将项目克隆至个人工作目录，结合自用浏览器插件，将 PaperBridge 首页设为新标签页，每日工作开始时快速浏览资源更新状态，提升文献调研效率。

3. **网络受限环境下的备用入口**：在部分校园网或企业内网环境下，主流学术站点可能访问缓慢或无法直达。PaperBridge 整理的镜像地址列表可为用户提供备选访问路径，通过尝试不同镜像源维持文献获取能力。

4. **开源信息学教学案例**：信息管理类或计算机科学相关课程可将 PaperBridge 作为数据组织与信息检索的实践案例，引导学生分析 URL 结构、设计分类体系，并编写自定义爬虫扩展资源列表。

5. **社区共建资源索引**：技术社区或开源兴趣小组可 fork 本项目，在小组内部维护一份定制化的资源清单，通过 Pull Request 互相补充有效链接，形成动态更新的知识图谱。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，确保已安装 Git 及 Python 3.6 以上版本。

```bash
# 克隆项目仓库至本地
git clone https://github.com/paperbridge/paperbridge-index.git
cd paperbridge-index

# 安装依赖（仅需 Python 标准库，无第三方包）
# 若系统缺少 python3-venv，可跳过虚拟环境，直接使用系统 Python
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate

# 启动本地静态服务器，默认监听 8000 端口
python3 -m http.server 8000
```

启动成功后，在浏览器中访问 `http://localhost:8000` 即可查看索引首页。若需更改端口，可在命令后追加端口号，例如 `python3 -m http.server 8080`。

## 安装要求

| 依赖项 | 必需性 | 说明 |
|---|---|---|
| Git 2.20+ | 必需 | 用于克隆仓库及后续拉取更新，低版本可能不支持部分协议 |
| Python 3.6+ | 必需 | 运行内置 HTTP 服务器及可用性检查脚本，3.6 以下版本缺少 f-string 支持 |
| 网络连通性（IPv4） | 必需 | 部分资源站点仅支持 IPv4，纯 IPv6 环境下可能无法访问 |
| 浏览器（Chrome / Firefox / Edge） | 推荐 | 查看索引页面的最佳体验，推荐使用最新版本 |
| curl 或 wget | 可选 | 用于命令行快速测试站点状态，非必需但便于调试 |
| make 或 gnumake | 可选 | 部分自动化脚本提供 Makefile 封装，非必需可手动执行 |

## 项目结构

```
paperbridge-index/
├── index.md                  # 主索引文件，包含全部资源表格及分类

├── config.yaml               # 用户自定义配置文件，可覆盖分组标签及备注
├── scripts/                  # 工具脚本目录
│   ├── check_availability.py # 批量 URL 可用性检查脚本，输出 CSV 报告
│   ├── generate_html.py      # 将 index.md 转换为独立 HTML 文件的构建脚本
│   └── utils.py              # 公共函数库，含日志、日期处理与网络请求封装
├── docs/                     # 详细文档目录
│   ├── quickstart.md         # 快速入门指南，含截图示例
│   ├── maintenance.md        # 日常维护流程，含链接增删改操作规范
│   ├── categories.md         # 分类法说明，含标签层级关系
│   └── troubleshooting.md    # 常见故障诊断与解决方案
├── tests/                    # 单元测试目录
│   ├── test_checker.py       # 可用性检查函数的单元测试
│   └── test_parser.py        # 配置文件解析与表格生成测试

```

## 贡献指南

我们欢迎社区贡献者以多种形式参与 PaperBridge 的改进与维护。请遵循以下步骤：

1. **Fork 仓库并创建功能分支**：从主仓库 fork 到个人账户，然后基于 `main` 分支创建新分支，分支命名建议使用 `feat/` 或 `fix/` 前缀，例如 `feat/add-physics-mirror`。

2. **更新资源列表并同步文档**：在 `index.md` 中按现有格式添加或移除链接，同时更新 `docs/categories.md` 中的分类说明，确保表格内 URL 无多余空格或换行。

3. **运行可用性检查脚本验证**：在提交前，执行 `python3 scripts/check_availability.py` 确保新增链接返回状态码为 200 或 30x，并检查输出报告中无异常报错。

4. **编写清晰的提交信息**：提交信息应包含简明摘要和详细描述，引用相关 issue 编号（如有）。示例格式：`docs: add new mirror site for condensed matter physics`。

5. **发起 Pull Request 并等待审核**：将分支推送至个人 fork，然后向主仓库的 `main` 分支发起 PR。PR 描述中需注明改动动机、测试结果及是否影响现有功能。审核周期通常为 3 个工作日。

## 常见问题

**问：部分链接无法访问，项目是否提供自动替换机制？**

答：项目本身不提供自动替换功能，因为涉及外部站点的可用性变化具有时效性。但我们建议用户定期运行 `scripts/check_availability.py` 生成不可达站点列表，并手动在 `index.md` 中标注或移除失效链接。社区也鼓励用户在 Issue 中报告失效站点，以便维护者协同更新。

**问：可以将 PaperBridge 用于商业产品或嵌入到其他系统吗？**

答：可以。PaperBridge 采用 MIT 许可证，允许自由使用、修改、分发，包括用于商业目的。但请注意，项目引用的外部资源站点各自拥有独立的版权和使用条款，使用者应自行遵守相关站点的服务协议，PaperBridge 项目及维护者不对第三方站点的内容合法性及可用性承担任何责任。

**问：本地服务器启动后，页面显示空白或样式丢失怎么办？**

答：请首先确认 `index.html` 文件是否与 `templates/base.tpl` 处于同一目录层级，且 `templates` 文件夹权限可读。若使用 `python3 -m http.server` 启动，确保当前工作目录为项目根目录，而非 `docs` 或 `scripts` 子目录。若问题仍未解决，可尝试重新执行 `python3 scripts/generate_html.py` 重新生成页面，然后刷新浏览器缓存（Ctrl+F5）。
