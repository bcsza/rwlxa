# LinkNest

LinkNest 是一个面向开发人员与技术内容消费者的高质量外链资源聚合平台。项目定位为“技术导航的导航”，通过人工筛选与自动化健康检查相结合的方式，持续维护一份覆盖影视、动漫、在线工具、开发文档等领域的优质外部资源清单。目标用户包括需要快速查找替代镜像站点的运维工程师、希望发现小众工具产品的产品经理，以及日常依赖多站点并行检索的技术研究人员。LinkNest 并不存储或托管任何第三方内容，仅提供客观、稳定、可验证的 URL 索引服务，解决用户在信息过载环境下“找到对的站点”比“找到站点”更困难的核心痛点。

## 功能概览

- **智能可用性探测**：系统每 6 小时对收录的全部外链执行一次 TCP 连通性与 HTTP 状态码检查，自动标记异常节点并发送告警通知。

- **分类标签体系**：每个资源条目支持多级分类标签，如“影视”、“动漫”、“工具”、“文档”、“镜像”等，支持按标签组合筛选。

- **用户提交通道**：开放社区提交入口，用户可通过 Pull Request 或 Issue 模板推荐新资源，经维护者审核后合并入主索引。

- **访问统计看板**：提供每个外链的点击量、最近 7 日可用率、平均响应时间等可视化指标，辅助判断资源质量。

- **离线快照缓存**：对于部分访问不稳定但内容有价值的站点，提供 24 小时内的 HTML 快照缓存（仅限静态页面），保障基础可读性。

- **RSS 订阅源**：提供资源变更通知的 RSS 订阅，包括新增、删除、状态变更等事件，便于自动化集成。

- **批量导出功能**：支持将当前索引库导出为 JSON、CSV 或纯文本列表格式，方便其他工具或脚本二次处理。

- **黑暗模式与无障碍适配**：前端界面遵循 WCAG 2.1 AA 标准，支持系统级深色主题自动切换。

## 应用场景

**日常技术调研中的快速备选**：当开发者需要查找某个知名工具或文档站点的可用镜像时，可在 LinkNest 中按标签或关键词检索，获得经过健康校验的备选列表，避免反复手动尝试域名变体。

**内容聚合站点的外链维护辅助**：中小型内容平台或导航站点的运营者，可定期参考 LinkNest 的索引库更新自身外链区域，减少死链率，提升用户体验。项目公开的导出接口支持定时拉取差异数据。

**网络环境受限场景下的资源发现**：对于处于特定网络环境中的用户，部分常规域名可能无法直接访问。LinkNest 维护的多种类镜像与替代站点列表，可帮助用户快速找到可访问的等效资源，提高信息获取效率。

**开源项目文档的外部引用校验**：开源项目的 README 或 Wiki 中常引用大量外部链接。维护者可使用 LinkNest 的批量检查功能，在版本发布前校验所有外部引用的可用性，降低用户按文档操作时的失败概率。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，需预先安装 Git、Node.js 18+ 与 pnpm。

```bash
# 克隆仓库
git clone https://github.com/your-org/linknest.git
cd linknest

# 安装依赖
pnpm install

# 启动开发服务器（默认占用端口 3000）
pnpm run dev
```

生产环境构建与静态导出：

```bash
pnpm run build
pnpm run export
```

导出产物位于 `out/` 目录，可直接部署至任意静态托管服务。

## 贡献指南

我们欢迎社区成员以多种形式参与 LinkNest 的建设，所有贡献者需遵守行为准则与贡献者许可协议。

**提交新资源**：通过 GitHub Issue 使用“资源推荐”模板填写资源名称、原始 URL、分类标签及简要描述。维护者将在 3 个工作日内审核，审核通过后由维护者合并入索引库。

**报告失效链接**：若发现索引中某个 URL 连续 3 次健康检查失败但页面实际可访问，请提交 Issue 说明情况并附上访问截图或 curl 输出，协助调整检查策略。

**改进文档或界面**：Fork 本仓库后创建功能分支，修改完成后提交 Pull Request。请确保所有提交通过 CI 检查（包括 ESLint、Prettier、TypeScript 类型检查及单元测试）。对于 UI 改动，需附上前后对比截图。

**开发新检测协议**：目前支持 HTTP/HTTPS 与 TCP 检测，若希望增加 DNS 解析检测、证书有效期检测或 WebSocket 检测，请先通过 Discussion 发起技术方案讨论，通过后再进行开发。

**运行测试与代码规范**：提交前请在本地执行 `pnpm run test` 和 `pnpm run lint`，确保无错误与警告。提交信息请遵循 Conventional Commits 格式。

## 常见问题

**Q：LinkNest 是否存储或代理第三方站点的内容？**

A：LinkNest 不存储任何第三方站点的音视频、图片或可执行文件。项目仅保存 URL 字符串及基础元数据（标题、标签、状态码）。离线快照功能仅对静态 HTML 页面进行文本级缓存，且不对外提供直接访问，仅供内部健康分析使用。

**Q：为什么某些资源 URL 没有 https:// 或 http:// 前缀？**

A：LinkNest 忠实记录用户提交时的原始 URL 格式。部分资源以裸域名形式提交，以保留用户原始使用习惯。实际访问时，前端会依据当前页面协议自动补全同协议前缀，但索引存储与导出始终保留原始字符串。

**Q：如何将我维护的资源列表整体迁移到 LinkNest？**

A：您可以使用批量导入接口。先将资源整理为 CSV 格式（列：name, url, category, tags），然后通过管理后台的“批量导入”功能上传，或直接调用 POST /api/resources/batch 接口。导入前请确保 CSV 编码为 UTF-8，且每行 URL 不包含多余空白字符。
