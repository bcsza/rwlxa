# LinkPilot

LinkPilot 是一个面向技术内容创作者与开源项目维护者的轻量级外链资源聚合与导航系统。该项目定位于解决技术文档、项目 README、博客文章及社区论坛中外部链接分散、难以统一管理与版本追踪的问题。LinkPilot 本身不存储任何第三方资源内容，仅提供结构化的链接分类、状态检测与展示框架，适用于需要频繁引用外部视频、图片、资讯站点或动态资源的技术内容聚合场景。

LinkPilot 的核心目标用户包括开源项目文档维护者、技术博客作者、社区运营人员以及需要定期整理外部参考链接的研发团队。通过统一的配置入口与清晰的分类视图，用户可以快速建立可维护、可审计的外链资源目录，减少因链接失效、域名变更或内容迁移带来的文档维护成本。

## 功能概览

- **多层级分类导航**：支持按资源类型、业务领域或使用频次对链接进行自定义分组，便于不同角色快速定位所需外链。

- **链接状态周期性检测**：内置轻量级 HTTP 状态检查器，可定期探测每个外链的可达性，并在管理界面中标注异常链接，辅助维护者及时更新或替换失效资源。

- **原始地址透传展示**：所有收录的外部链接均以原始字符串形式直接呈现，不进行任何协议补全、域名规范化或 URL 重写，确保地址与用户提供的完全一致，避免引入额外跳转或解析偏差。

- **批量导入与导出**：支持通过纯文本或结构化配置文件批量导入链接清单，并可导出为 Markdown 表格或纯文本列表，方便嵌入项目文档或迁移至其他系统。

- **标签与全文检索**：为每条链接附加可搜索的标签与简要描述，支持基于标签组合的快速过滤，提升大量链接下的查找效率。

- **访问统计与点击追踪**：可选开启本地点击日志记录，统计各外链的引用次数与最近访问时间，帮助维护者了解资源的实际使用热度。

- **权限分级管理**：提供只读浏览、链接编辑、系统配置三级权限，适合多人协作维护的场景，避免误删或误改生产环境链接配置。

## 应用场景

- **开源项目外部依赖索引**：当项目文档中引用了大量第三方教程、视频演示、图片素材或行业报告时，维护者可使用 LinkPilot 集中管理这些外链，并在 README 或官网中嵌入只读导航页面，确保文档简洁且链接可追溯。

- **技术社区资源周报整理**：社区运营人员可将本周讨论中提及的热门视频站点、新闻链接或工具地址统一收录至 LinkPilot，生成周报外链附录，供社区成员后续查阅，同时避免在论坛帖子中堆叠冗长 URL。

- **个人知识库外链备份清单**：技术博主或研究员可将写作过程中参考的视频、图片、资讯等外部资源统一录入系统，定期检测可用性，防止关键引用素材因源站调整而永久丢失。

- **多项目共享资源目录**：企业内多个产品线可共用一套 LinkPilot 实例，按项目标签隔离各自的外链集合，同时共享公共技术文档、设计规范或培训视频链接，减少重复整理工作。

## 快速开始

以下步骤帮助您在本地环境快速启动 LinkPilot 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkpilot/linkpilot.git

# 进入项目目录并安装依赖（使用 npm）
cd linkpilot
npm install

# 复制示例配置文件并调整本地参数
cp .env.example .env

# 初始化本地 SQLite 数据库结构
npm run migrate

# 启动开发服务器（默认监听 3000 端口）
npm run dev
```

启动成功后，访问控制台输出中显示的本地地址（通常为 `http://localhost:3000`）即可进入 LinkPilot 管理界面。首次启动将自动创建默认管理员账户，初始密码见启动日志提示，请及时修改。

1. 在 GitHub 上 fork 本仓库至个人账号，并克隆到本地开发环境中。建议使用 `dev` 分支作为基础，避免直接修改 `main` 分支。

2. 新建功能分支并实现您的改进或修复。提交前请确保代码通过所有现有单元测试，并为新增功能补充对应的测试用例，测试覆盖率不低于 80%。

3. 编写或更新相应的文档内容，包括但不限于 README 中受影响的部分、配置说明或 API 注释。文档变更需与代码变更一同提交。

4. 提交 pull request 至本仓库的 `main` 分支，并在 PR 描述中清晰说明变更目的、影响范围以及测试验证情况。PR 将由项目维护者进行代码审查，审查通过后合并。

5. 若您希望长期参与核心模块开发，可联系维护者申请加入组织成员，获得直接推送权限，但仍需遵守分支管理与评审流程。

## 常见问题

**Q: LinkPilot 是否支持 PostgreSQL 或 MySQL 作为生产数据库？**

A: 当前版本默认使用 SQLite 作为存储引擎，主要面向中小规模部署。若您需要企业级高并发场景，可以通过修改 `config/default.json` 中的数据库连接字符串切换至 PostgreSQL，但需自行安装对应驱动并调整迁移脚本。社区版暂不提供官方对其他数据库的完整测试与支持。

**Q: 链接状态检测是否会误报或影响源站访问？**

A: 检测器默认使用 `HEAD` 请求，仅获取响应头信息，不会下载页面内容，对源站负载影响极小。检测超时时间设为 5 秒，并支持配置重试策略。对于部分严格限制请求频率的站点，您可以在配置中增加该域名的检测间隔或将其加入白名单跳过检测。
