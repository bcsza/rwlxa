# LinkPilot

LinkPilot 是一个面向开发者和技术内容创作者的开源外链资源聚合与导航工具。它旨在解决个人或团队在维护项目文档、技术博客、社交媒体主页时，分散收藏、难以同步更新、链接失效等资源管理痛点。通过结构化的数据配置和静态站点生成逻辑，LinkPilot 帮助用户将零散的高价值外链转化为清晰、可维护、支持快速检索的资源目录。

该项目适用于需要频繁引用外部技术文档、API 参考、视频教程或社区论坛的开发者，也适用于希望构建轻量级行业资源导航页的技术团队。LinkPilot 不依赖数据库，所有资源数据以 Markdown 或 YAML 格式存储，便于版本控制和自动化部署。其核心价值在于将「收藏」行为升级为「可管理资产」，降低长期维护成本，提升信息复用效率。

## 功能概览

- **多级分类与标签系统**：支持无限层级的资源分类和灵活的打标签机制，允许同一资源归属于多个分类或标签，便于从不同维度检索。

- **链接健康状态检测**：内置定时检测模块，可对已收录的 URL 进行可达性检查，自动标记状态码异常或响应超时的链接，并在管理面板中高亮提示。

- **全文本模糊搜索**：基于倒排索引实现轻量级本地搜索，支持对资源标题、描述、标签、分类名称进行关键词匹配，搜索结果按相关度排序。

- **静态站点一键导出**：提供 `build` 命令，将当前资源数据渲染为纯静态 HTML 页面，输出产物可直接部署至 Nginx、CDN 或 GitHub Pages，无需额外运行时环境。

- **数据导入与导出**：支持从 CSV 文件批量导入链接，导出格式包括 JSON、YAML 和 Markdown 表格，便于与其他工具链集成或进行数据备份。

- **访问量统计与热度排序**：记录每个外链在本站内的点击次数，支持按点击量、添加时间、名称字母序进行动态排序，帮助识别高频使用资源。

- **自定义 CSS 与模板覆盖**：允许用户通过替换 `theme/` 目录下的样式文件和模板片段，完全自定义前端展示风格，满足品牌化或个性化需求。

- **多用户协作权限控制**：提供基于角色的访问控制（Admin/Editor/Viewer），支持多人协同维护资源列表，操作日志可追溯。

## 应用场景

1.  **技术团队内部文档中心**：开发团队可将日常使用的框架官方文档、依赖库主页、CI/CD 工具地址、内部监控面板等统一收录至 LinkPilot，作为团队知识库的导航页，减少新成员上手时的信息查找时间。

2.  **开源项目 README 外链托管**：开源维护者通常需要在 README 中放置大量参考链接，但 README 篇幅有限且更新不便。使用 LinkPilot 生成独立外链页面，在 README 中仅放置一个主入口链接，即可承载全部扩展阅读材料，同时利用健康检测功能及时发现失效引用。

3.  **技术博客或自媒体资源推荐**：技术博主或视频 UP 主可以在博客侧边栏或视频简介中嵌入 LinkPilot 托管的资源目录，定期整理推荐工具、书籍、教程站点，提升内容附加值，并借助访问统计了解读者偏好。

4.  **黑客松或编程竞赛辅助平台**：比赛组织方可将赛题相关数据源、API 文档、示例代码仓库、社区讨论区链接集中发布，参与者在一个页面内获取全部所需外部资源，减少因分散查找导致的时间浪费。

5.  **个人开发者的知识收藏夹**：独立开发者可使用 LinkPilot 管理个人长期积累的技术收藏，按项目阶段（学习、设计、编码、测试、部署）分类，并定期运行检测清理失效链接，保持收藏夹的鲜活度。

## 快速开始

以下命令序列可在本地快速启动 LinkPilot 开发实例，默认使用 SQLite 数据库并监听 8080 端口。

```bash
# 克隆主仓库
git clone https://github.com/your-org/linkpilot.git
cd linkpilot

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并导入示例数据，启动开发服务器
python manage.py migrate
python manage.py loaddata fixtures/sample_links.json
python manage.py runserver 0.0.0.0:8080
```

启动成功后，访问 `http://localhost:8080` 即可浏览示例资源库。管理员后台路径为 `/admin`，默认账号 `admin` 密码 `linkpilot2024`（首次登录后强制修改）。

## 常见问题

**Q: 健康检测模块误报或漏报怎么办？如何调优？**

A: 检测模块默认使用 `curl -I -m 5` 获取响应头，超时时间 5 秒，重试 1 次。对于某些响应缓慢但实际可用的站点（如部分境外服务），可在 `config/settings/base.py` 中修改 `HEALTH_CHECK_TIMEOUT` 和 `HEALTH_CHECK_RETRIES` 变量。如需排除特定 URL 的检测，可在管理后台将该链接的 `check_enabled` 字段取消勾选。若检测结果与人工访问不一致，请检查网络环境是否配置了代理，或目标站点是否启用防爬机制返回 403。

**Q: 静态导出生成的页面无法正确显示搜索功能？**

A: 静态导出模式（`build` 命令）不包含服务端搜索接口，因此搜索框在静态页面中默认隐藏。若需在静态部署中保留搜索能力，可选择使用外部搜索引擎（如 Algolia 或 Pagefind）进行索引，并将搜索框重定向至第三方搜索页面。具体配置方式请参考 `docs/deployment/static_build.md` 中的「外部搜索集成」章节。

**Q: 如何从 SQLite 迁移至 PostgreSQL 以承载更大规模的链接数据？**

A: 项目支持数据库无缝切换。首先在 `requirements.txt` 中添加 `psycopg2-binary` 依赖并重新安装。然后在生产配置中修改 `DATABASES` 字典的 `ENGINE` 为 `django.db.backends.postgresql`，并提供 `NAME`、`USER`、`PASSWORD`、`HOST`、`PORT` 参数。数据迁移可使用 `python manage.py dumpdata --exclude auth.permission --exclude contenttypes > backup.json` 导出，切换数据库配置后执行 `python manage.py migrate` 创建表结构，最后用 `python manage.py loaddata backup.json` 恢复数据。注意外键约束和自增主键的冲突处理，建议在维护窗口执行。

**Q: 能否在 LinkPilot 中嵌入视频或 iframe 内容？**

A: LinkPilot 定位为外链导航工具，默认不支持内嵌 iframe 或视频播放器。但您可以在资源描述的 Markdown 字段中直接添加标准的 HTML 视频标签或 iframe 代码，前端模板默认会渲染通过 `safe` 过滤器标记的内容。出于安全考虑，此功能仅在管理员编辑视图中开放，普通访客无法提交此类内容。
