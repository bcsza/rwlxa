# Tomato Index

Tomato Index 是一个面向数字文化研究者、媒体监测机构与内容爱好者的高时效性网络资源导航与元数据聚合项目。项目不存储、不分发、不缓存任何第三方内容，仅提供公开可访问的网络资源入口索引与基础元信息描述，帮助用户快速定位特定领域的优质信息节点。

项目定位为轻量级、可自部署的资源外链汇总站，适用于需要系统性梳理特定网络空间信息分布、追踪热点域名变化趋势、或构建自定义网络资源看板的场景。Tomato Index 本身不涉及爬虫、数据采集或自动化扫描，所有收录的资源均基于人工筛选与公开信息整理，并以静态站点形式提供高速访问能力。

## 功能概览

- 资源分类索引 提供按内容主题、服务类型、活跃状态等多维度分类的资源导航目录，支持快速筛选。

- 域名状态快照 记录收录资源的可用性状态与基础响应信息，辅助判断资源的当前服务情况。

- 元信息标注 每个资源条目附带收录时间、内容描述、语言、国家/地区归属等元数据，便于信息检索与归档。

- 自定义标签系统 用户可为资源添加自定义标签，构建个性化的分类体系，并支持标签检索与聚合。

- 全文检索 集成轻量级搜索功能，支持对资源名称、描述、标签、域名等字段进行快速关键词匹配。

- 批量导入导出 支持通过 CSV 或 JSON 格式批量导入资源列表，并支持导出为结构化数据文件，便于迁移或二次处理。

- 访问量统计 内置简单的资源点击计数与热门排行功能，帮助识别高频访问的资源节点。

- 响应式管理面板 提供适配移动端与桌面端的管理后台，支持资源的增删改查操作，适合个人或小团队维护。

## 应用场景

- 数字档案维护 研究者或档案馆可利用 Tomato Index 整理特定主题下的网络资源分布，形成结构化的公开信息索引，辅助长期追踪与存档。

- 内容聚合站点前置层 内容平台或资讯站点可将 Tomato Index 作为外部资源引用的前置导航层，替代硬编码链接，提升资源维护效率与灵活性。

- 个人知识管理 技术爱好者或信息收集者可自部署 Tomato Index 作为个人网络书签的增强版本，利用标签和元数据功能实现更精细化的资源分类。

- 内部协作工具 团队内部可利用 Tomato Index 统一管理项目相关的第三方参考资源，通过共享索引减少信息孤岛，提升协作效率。

- 舆情监测辅助 媒体监测机构可将 Tomato Index 作为快速访问特定信息源的入口集合，结合人工研判进行定向信息追踪。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git、Node.js 18.x 及以上版本、pnpm 8.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/tomato-index/tomato-index.git
cd tomato-index

# 安装项目依赖
pnpm install

# 启动开发服务器，默认访问 http://localhost:5173
pnpm run dev
```

生产环境构建与部署：

```bash
# 构建静态站点
pnpm run build

# 预览构建结果
pnpm run preview
```

构建产物默认输出至 dist 目录，可部署至任意静态托管服务（如 Nginx、Caddy、Vercel、Netlify 等）。

## 贡献指南

项目欢迎各类形式的贡献，包括但不限于新增资源收录建议、文档改进、Bug 修复、功能增强等。请遵循以下流程以保证协作效率：

1. 提出问题或建议 先在 Issue 列表中检索是否已有类似讨论，若无，则新建 Issue 并详细描述您的需求或发现的问题，提供复现步骤或改进依据。

2. 分支开发 若计划提交代码变更，请从 main 分支新建功能分支（feat/xxx 或 fix/xxx），确保分支名称简洁描述变更内容。开发过程中请保持代码风格与项目现有规范一致。

3. 编写测试与文档 对于新增功能或修复，请补充相应的单元测试或端到端测试用例，并同步更新 docs 目录下的相关文档，确保文档与代码保持同步。

4. 提交 Pull Request 完成开发后，将分支推送至仓库并提交 Pull Request，在描述中引用关联的 Issue 编号，并简要说明变更内容与影响范围。项目维护者将在 3 个工作日内进行审核。

5. 审核与合并 审核通过后，由项目维护者执行合并操作。合并后 CI 流水线将自动触发构建与部署，确保主分支始终保持可发布状态。

## 常见问题

Q: Tomato Index 是否存储或缓存第三方资源的内容？

A: 不存储。本项目仅收录资源入口的 URL 地址及其人工标注的元数据，不涉及任何内容抓取、代理转发或缓存服务。所有对第三方资源的访问均直接由用户浏览器发起，与本项目无内容传输关系。

Q: 如何更新已收录资源的可用性状态？

A: 项目提供了手动状态标注功能，管理员可在管理后台对每个资源进行状态标记（如正常、异常、失效、迁移中）。当前版本未集成自动化探测机制，以避免产生不必要的网络请求，确保项目轻量合规。用户可定期人工复核并更新状态。

Q: 部署后首次启动显示空白页面该如何处理？

A: 此情况通常由环境变量缺失或依赖安装不完整导致。请检查 .env 文件是否存在并包含必要的配置项（如 API_BASE_URL），并重新执行 pnpm install 确保所有依赖正常安装。若问题仍存在，可查看浏览器开发者工具控制台输出的错误信息，并参考 /docs/operations/troubleshooting.md 进行排查。
