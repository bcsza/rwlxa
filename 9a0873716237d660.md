# HyperLink Harvest

HyperLink Harvest 是一个面向数字媒体从业者、技术研究者及信息分析人员的结构化网络资源导航与状态监测工具。本项目并非传统意义上的网址收藏夹，而是一套可工程化部署的链接资产管理与可用性观测系统。其核心定位在于帮助用户从分散、易失效的网络入口中抽取出稳定的信息通道，并通过定期巡检与元数据标注，降低因链接变动导致的信息获取成本。

目标用户包括：需要持续追踪特定领域内容更新的研究人员、运维多站点内容分发的小型团队、以及希望对自身网络书签进行技术化治理的高级用户。HyperLink Harvest 不生成内容，也不聚合内容，而是提供一种轻量级的链接生命周期管理方案，将原始 URL 资源转化为可分析、可追溯、可预警的数据资产。

## 功能概览

- **原始链接归档**：以不可变方式存储用户提交的原始 URL 及其元数据，包括提交时间、来源批次与初始状态标记，确保每一个链接的原始形态可被精确回溯。
- **可达性周期性检测**：支持自定义时间间隔的 HTTP/HTTPS 探活检查，自动记录状态码、响应时间与重定向链，并生成可用性趋势图表。
- **变更指纹追踪**：对目标资源返回的响应头关键字段与内容摘要进行哈希计算，及时发现页面结构或版本信息的非预期变动。
- **分类标签与全文检索**：允许用户为每条链接添加多维度标签（如地域、语种、内容主题），并基于标题、描述及标签进行快速过滤与全文搜索。
- **状态预警与报告输出**：当链接连续不可达或响应时间超过阈值时，通过 Webhook 或邮件发送告警；同时支持按周、月维度生成可用性统计报告。
- **批量导入与导出**：支持 CSV、JSON 及纯文本列表格式的链接批量导入，并可导出为结构化数据文件，便于与其他数据看板或监控系统集成。
- **用户权限与团队协作**：提供基于角色的访问控制，支持多成员协同管理链接库，并记录所有操作审计日志。

## 应用场景

- **媒体资源渠道稳定性监控**：内容聚合平台可利用本系统定期检测第三方素材站点的连通性，当某个图片来源或视频源站出现访问异常时，系统自动发出预警，避免内容展示空窗。
- **跨境业务网络入口治理**：面向海外市场的运营团队常需维护大量不同地区的服务入口。HyperLink Harvest 可按照地理标签分类管理这些 URL，并分别检测各区域解析与访问延迟，辅助调度决策。
- **开源项目依赖资源备份**：开源社区维护者可使用本工具跟踪项目文档、二进制分发地址及镜像站点的可用性，当上游链接失效时，及时切换至备用源，保障用户下载体验。
- **个人信息库数字化整理**：高级技术用户可将长期积累的收藏夹、笔记中的外链导入系统，利用变更追踪功能感知页面重要信息更新，避免错过关键公告或版本发布。

## 快速开始

以下步骤将引导您在本地环境中快速启动 HyperLink Harvest 服务。

```bash
# 克隆代码仓库
git clone https://github.com/hyperlink-harvest/harvest-core.git
cd harvest-core

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动调度器与 Web 服务
python manage.py migrate
python manage.py init_superuser --username admin --email admin@example.com
python manage.py runserver --host 0.0.0.0 --port 8080 --enable-scheduler
```

访问 http://localhost:8080 即可进入 Web 控制台。默认管理员密码将在初始化时输出至终端，首次登录后请立即修改。

## 常见问题

**Q：巡检任务是否会对目标网站造成压力？如何控制？**
A：HyperLink Harvest 默认采用指数退避策略与随机抖动，避免在同一时间窗口内对同一域名发起密集请求。用户可在巡检策略配置中设置单任务并发度（默认 5）、请求间隔（最小 2 秒）以及自定义 User-Agent。对于敏感目标，建议手动配置白名单或降低巡检频率。

**Q：系统如何处理链接重定向或内容迁移？**
A：系统默认跟随重定向（最多 5 次），并记录最终响应 URL 与状态码。若最终 URL 与原始 URL 域名不一致，系统会标记为“重定向跨域”并保留两条记录，便于用户识别长期迁移。用户也可在巡检规则中关闭跟随重定向，仅检测初始请求的响应。

**Q：能否将巡检数据导出至第三方监控平台（如 Prometheus）？**
A：可以。系统提供 `/metrics` 端点，以 Prometheus 格式暴露关键指标（链接可用率、平均响应时间、任务队列深度）。用户可在 `alert` 模块中配置 Webhook 地址，将告警事件实时推送至外部事件网关。
