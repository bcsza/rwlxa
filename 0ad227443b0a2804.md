# Tangram Navigator

Tangram Navigator 是一个面向中文互联网用户的轻量级技术资源导航与聚合站。项目定位为“结构化外链枢纽”，不存储任何实体媒体文件，仅提供公开可访问的 URL 索引与分类整理服务。目标用户包括个人开发者、技术内容创作者、资源收集爱好者以及需要快速检索特定类型网站链接的普通网民。项目解决的核心痛点是：大量优质垂直领域域名分散在社交平台零散帖子中，缺乏统一、可维护、低延迟的索引机制，用户每次检索需重复进行低效搜索操作。

项目采用纯静态 Markdown 渲染方案，所有链接数据以配置化方式维护，支持一键生成站点地图，便于搜索引擎爬取。同时提供面向人工阅读的分类视图和面向机器解析的 JSON 导出接口，兼顾可读性与自动化集成需求。Tangram Navigator 不依赖后端数据库，可部署于任何支持静态托管的 CDN 或对象存储服务，启动时间小于 200 毫秒，单页链接数承载上限超过 5000 条。

## 功能概览

- **按领域分类索引**：将收集到的 URL 按影视、动漫、纪录片、公开课、技术社区等维度进行一级分类，每个分类下再按语种、清晰度、更新频率等次级标签筛选，方便用户快速缩小范围。

- **实时可用性探测**：集成轻量级 HEAD 请求检查模块，对每条外链进行每日三次的可用性探活，并在前端以颜色标识（绿/黄/红）呈现当前站点响应状态，减少用户点击无效链接的挫败感。

- **自定义标签系统**：允许用户为任意链接添加自定义标签（如“无广告”、“高速”、“需代理”），标签数据存储在浏览器本地 LocalStorage 中，实现个性化过滤而不影响全局索引。

- **模糊搜索与拼音补全**：搜索框支持中文分词、拼音首字母匹配及编辑距离容错（阈值设为 2），例如输入“zy”可匹配“资源”、“中原”、“竹影”等词条，显著提升移动端输入效率。

- **一键导出收藏夹**：提供导出功能，将当前筛选或搜索得到的链接列表以 CSV 或 Markdown 表格形式下载，便于用户迁移至其他书签管理工具或分享给团队成员。

- **更新日志聚合**：在侧边栏展示最近 30 天内新增或状态变更的链接条目，按时间倒序排列，帮助老用户快速感知资源库变化，避免重复浏览旧内容。

- **暗色主题与阅读模式**：内置两套配色方案（日间/暗色），并针对视频类链接提供“纯净阅读”模式，自动隐藏广告类外链（基于域名黑名单匹配），提升沉浸式浏览体验。

## 应用场景

- **影视资源搜集者日常检索**：用户需要查找某部冷门纪录片或早期国产电视剧的在线观看入口时，可直接在 Tangram Navigator 的“影视-经典”分类下按出品年份筛选，配合可用性探测结果，优先选择当前响应正常的站点，避免逐个尝试失效链接。

- **技术社区内容聚合与分享**：技术博主在撰写“2026 年值得关注的独立开发者资源”类文章时，可使用本项目的导出功能，按标签筛选出“开发者工具”、“设计素材”、“API 测试”等子集，生成干净的外链列表直接嵌入博文附录，节省手动整理时间。

- **网络运维人员链路质量抽查**：运维工程师可通过项目提供的 JSON 接口定时拉取全部链接列表，结合自建的监控系统对这批域名的 DNS 解析时长、TLS 握手耗时进行二次探测，用于评估不同运营商的访问质量差异，辅助 CDN 调度决策。

- **教育机构教学素材索引**：高校教师或 MOOC 助教可依据“公开课”和“学术数据库”分类，快速整理出本学期课程所需的参考视频链接与论文预印本站点，并将筛选结果导出为 CSV 文件分发给学生，避免学生自行搜索时误入钓鱼站点。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL2 环境，要求已安装 Git、Node.js 18.x 及以上版本、pnpm 8.x 及以上版本。

```bash
# 克隆仓库
git clone https://github.com/tangram-navigator/navigator.git
cd navigator

# 安装依赖（使用 pnpm 加速）
pnpm install

# 构建静态站点并启动本地预览服务（默认端口 4173）
pnpm run build
pnpm run preview
```

执行完毕后，打开浏览器访问 `http://localhost:4173` 即可查看本地渲染结果。若需修改链接数据源，请编辑 `src/data/links.json` 文件，遵循已有 JSON Schema（参见 `docs/schema.md`），重新运行 `pnpm run build` 生成更新后的页面。

## 贡献指南

1.  **查阅现有议题与项目板**：在提交新链接或功能请求前，请先浏览 GitHub Issues 及 Projects 看板，确认是否已有相似提议。若存在，可在该议题下补充评论而非重复创建，以减少维护者冗余沟通成本。

2.  **本地数据修改与自测**：Fork 本仓库后，在 `src/data/links.json` 中按 JSON Schema 追加或修改链接条目。随后运行 `pnpm run test` 执行所有单元测试与校验脚本，确保新增数据不破坏结构完整性。对于新增分类，需同步更新 `categories.json` 及对应的显示名称国际化映射。

3.  **提交变更说明**：Commit Message 请遵循 Conventional Commits 规范（如 `feat: add new movie link under classic category` 或 `fix: correct domain for yuyetv.shop`）。若为链接状态变更（如域名失效），请在正文中简要说明探测结果来源（例如“连续 7 天 HEAD 请求超时”）。

4.  **发起 Pull Request**：PR 标题应简洁描述核心变更，内容中需附带自测截图或命令行输出日志，证明本地构建通过且可用性探测模块对该域名的检查结果符合预期。PR 至少需要一位维护者 Approve 后方可合并。

5.  **遵守行为准则**：本项目遵循 Contributor Covenant 2.1 版本行为准则，所有讨论与代码审查过程需保持专业、尊重且具有建设性。严禁提交包含恶意代码、钓鱼链接或侵犯版权内容的条目。

## 常见问题

**Q1：为什么某个链接显示为红色（不可用），但我手动访问浏览器却能打开？**

A：可用性探测模块默认使用 HEAD 请求方法，仅检查 HTTP 状态码是否在 200-399 范围内，且响应时间小于 5 秒。部分站点可能屏蔽 HEAD 请求或强制返回 405 Method Not Allowed，但 GET 请求正常。此时探测器会标记为不可用，但您仍可手动点击访问。若您确认该站点长期稳定可访问，可在 GitHub Issues 中提交该域名及备注信息，维护团队将评估是否将该域名加入 HEAD 请求白名单，或调整为 GET 请求探测。

**Q2：我可以将自己维护的链接列表导入到 Tangram Navigator 中吗？**

A：可以。您可通过导出功能下载 CSV 模板，按照模板格式填充您自己的链接数据（必须包含 `title`、`url`、`category` 三列），然后在网页底部“导入”区域选择该 CSV 文件。导入的数据仅保存在您浏览器的 LocalStorage 中，不会上传至任何服务器，且会与全局索引合并显示，但带有“本地”标识以作区分。若您希望将您的列表合并至官方索引库，请按照贡献指南发起 Pull Request。

**Q3：项目是否会存储用户的浏览历史或点击记录？**

A：不会。Tangram Navigator 所有代码在前端运行，不部署任何后端日志收集服务。可用性探测结果缓存在浏览器 IndexedDB 中，仅用于减少重复网络请求。导出功能生成的文件完全由用户本地控制，项目不发送任何信标（Beacon）或埋点脚本。您可通过浏览器开发者工具的网络面板验证所有请求仅针对您点击的目标域名，而无任何第三方分析域名。
