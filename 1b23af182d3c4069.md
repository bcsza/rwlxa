# WebMirror Resource Aggregator

WebMirror Resource Aggregator 是一个面向技术调研、内容归档与网络资源可访问性监测的开源外链汇总工具。项目定位于为开发者、数据分析师及内容研究者提供结构化的媒体与资讯类域名清单，并通过自动化检测脚本辅助判断各资源的可用状态与响应特征。

本项目不提供任何实质性的内容托管或代理服务，仅作为公开可访问 URL 的元信息整理与状态探测框架。目标用户包括网络质量监测工程师、SEO 研究人员、以及需要批量管理外部链接渠道的运维人员。通过本项目提供的基线配置与检测脚本，用户可快速建立自有资源库的健康检查流水线。

## 功能概览

- **批量域名归集**：支持按批次导入原始 URL 列表，自动去重并按顶级域分类，生成标准化的资源索引文件。

- **可用性探测引擎**：内置轻量级 HTTP 探针，支持配置超时阈值、重试次数及预期状态码，输出结构化 JSON 格式的探测报告。

- **响应指标采集**：记录每次请求的响应时间、内容长度、响应头关键字段（Server、Content-Type），为性能分析提供基础数据。

- **变更通知钩子**：支持在资源状态发生变更（如从可用变为不可用）时触发自定义脚本，便于接入告警系统。

- **数据导出适配**：提供 CSV 与 Markdown 表格两种导出格式，方便嵌入周报、监控看板或文档站点。

- **定时任务集成**：通过 cron 表达式调度检测任务，支持单次执行与后台常驻两种运行模式，适配容器化部署场景。

- **日志分级记录**：按 INFO、WARN、ERROR 级别输出运行日志，支持文件回滚与标准输出同步，便于问题追溯。

## 应用场景

1. **外部链接健康度巡检**：运营人员可配置每日定时任务，扫描资源列表中所有域名的可达性，自动标记连续失败超过阈值的条目，并生成待清理清单。

2. **地区性网络策略模拟**：通过在不同地理位置的服务器上部署检测脚本，对比同一域名在不同区域的解析结果与访问延迟，辅助 CDN 调度决策。

3. **历史数据归档质量验证**：内容归档系统可调用本项目的探测接口，在入库前验证目标链接的有效性，降低无效数据占用的存储成本。

4. **开源情报关联分析**：安全研究人员可将探测结果与域名注册信息、IP 归属地数据交叉比对，识别潜在的基础设施关联模式。

## 快速开始

以下命令适用于 Linux/macOS 环境，确保系统已安装 Git 与 Python 3.10 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/webmirror/resource-aggregator.git
cd resource-aggregator

# 安装依赖（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行检测引擎（默认加载 data/seeds_41.json）
python probe.py --batch 41 --output reports/status_41.json
```

## 常见问题

**问：探测过程中出现大量连接超时，应如何调整参数？**

答：可通过修改 `config/settings.yaml` 中的 `timeout` 字段（单位秒）适当延长等待时间，并调整 `max_retries` 值增加重试次数。若目标域名存在严格的访问频率限制，建议同时调低 `concurrency` 参数避免触发防火墙策略。

**问：如何排除某些 URL 不参与状态变更告警？**

答：在批次种子文件（如 `batch_41.json`）中为特定条目添加 `"alert": false` 字段，探测引擎在处理时会跳过该条目的告警钩子触发。该字段仅用于控制告警行为，不会影响探测数据采集与报告输出。

**问：能否将探测结果自动推送至外部监控面板？**

答：项目提供 `reporter.py` 的扩展接口，用户可通过继承 `BaseExporter` 类实现自定义导出器，例如写入 InfluxDB、Prometheus Pushgateway 或 Elasticsearch。具体实现参考 `docs/operation.md` 中的自定义导出章节。
