# WushuTV Navigator

WushuTV Navigator 是一个精心整理的技术资源导航与外部链接聚合系统，专为开发人员、运维工程师和技术研究者设计。本项目的核心目标在于解决技术社区中优质资源分散、官方文档入口难以快速定位、以及镜像站与备用域名信息不对称等实际问题，通过结构化的资源收录机制，为中文技术用户提供一个稳定、可扩展、可维护的起点页与服务发现层。

本项目并非传统的内容聚合平台，而是一个轻量级的链接治理与分发枢纽。它不对原始资源进行二次加工或代理，而是以人工精选与社区反馈相结合的方式，持续维护一份高质量的外部链接清单，覆盖流媒体工具、技术文档、开发社区、网络诊断、系统运维等多个垂直领域。WushuTV Navigator 适用于个人开发者搭建本地知识库入口、小型团队统一技术栈书签、以及大型组织内部分发可信外部参考资源等场景。项目设计上强调透明度、可审计性与低维护成本，所有链接变更均通过版本控制系统记录，并附带变更说明与状态标记。

## 功能概览

- **精选技术资源索引**：收录超过 200 个经过人工核验的优质外部链接，按功能域与使用频次分级标注，方便快速检索与日常复用。

- **多维度分类体系**：资源按照协议类型、服务性质、运维层级、社区活跃度等多个维度进行交叉分类，支持按标签筛选与自定义视图切换。

- **状态监控与可用性标记**：对每个收录的 URL 进行定期的连通性与响应时间检测，在列表中直观显示健康状态与历史可用率，辅助用户判断备用入口。

- **私有化部署支持**：提供完整的静态站点生成方案，可一键导出为纯 HTML 文件或 Docker 镜像，满足内网隔离环境下的资源分发需求。

- **社区反馈与更新机制**：内置 Issue 模板与 PR 校验流程，允许社区成员提交新增链接、报告失效地址或建议分类调整，所有变更经过审核后合并。

- **版本化变更日志**：所有链接的新增、移除、地址变更均记录在 CHANGELOG 中，支持按时间范围回溯，确保资源演进的透明性与可追溯性。

- **多语言入口适配**：针对中文用户常用的开发文档、API 参考、社区论坛等资源，提供语言标签与区域化推荐，降低跨语言检索成本。

## 应用场景

- **新项目技术选型时的参考入口**：当团队启动新服务或重构现有系统时，开发者可通过本项目快速访问各类运行时环境、数据库驱动、消息队列客户端、监控面板等官方文档与社区最佳实践，避免因重复搜索而浪费时间。

- **运维故障排查期间的快速链接跳转**：在系统异常或网络中断时，运维人员可通过本地部署的导航页面，一键跳转至各大云厂商状态页、DNS 诊断工具、网络延迟测速节点、日志分析平台等关键资源，缩短故障定位时间。

- **技术培训与新人入职引导**：团队可将本项目作为新员工 onboarding 资料的一部分，统一提供内部规范文档、外部参考手册、常用工具站点、代码仓库镜像等链接集合，减少信息差与学习曲线。

- **个人知识库与书签管理替代方案**：开发者可将本项目作为个人浏览器的起始页或新标签页替代方案，通过本地维护一份私有 fork 来管理个人高频使用的技术站点，同时保留社区公共更新带来的额外收益。

- **离线环境下的资源分发基准**：在物理隔离或网络受限的环境中，本项目可作为资源清单的基准文件，配合离线下载脚本与校验和文件，实现外部资源的有序导入与版本管理。

## 快速开始

以下步骤适用于 Linux 与 macOS 系统，Windows 用户可使用 WSL 或 Git Bash 环境执行。

```bash
# 克隆代码仓库至本地
git clone https://github.com/wushutech/navigator.git
cd navigator

# 安装依赖（基于 Node.js 22 LTS 与 pnpm）
corepack enable
pnpm install

# 构建静态站点并启动本地预览服务
pnpm build
pnpm preview --port 8080
```

执行完毕后，在浏览器中访问 http://localhost:8080 即可查看导航站点首页。若需要自定义资源列表，请编辑 `data/sources.json` 文件并重新执行构建命令。

## 贡献指南

1. 首先在 GitHub 上 fork 本仓库至个人账户，然后克隆到本地开发环境。请确保本地 Git 配置了正确的 user.name 与 user.email，以便提交记录可溯源。

2. 在 `data/sources.json` 中按照既定 JSON Schema 添加或修改链接条目。新增条目必须包含 `url`、`title`、`category` 和 `status` 字段，`status` 初始值建议设为 `unverified`。若为镜像站或备用地址，请在 `mirror_of` 字段中指明主站 ID。

3. 执行本地校验命令 `pnpm run validate` 以确保数据格式正确，随后运行 `pnpm run test` 确认所有单元测试与集成测试通过。若测试失败，请根据错误提示调整数据或代码。

4. 提交变更时，请使用语义化提交信息格式，例如 `feat: add new streaming source wushutv` 或 `fix: update xiniu88 endpoint to new domain`。提交信息应清晰描述变更内容与原因。

5. 推送分支至远程仓库后，通过 GitHub 界面创建 Pull Request。PR 描述中请勾选对应的变更类型，并附上链接可用性的简单验证结果（如 curl 输出或浏览器截图）。等待维护者审核，审核通过后即可合并入主分支。

## 常见问题

**问：我可以完全离线使用这个导航站吗？是否需要定期联网更新？**

答：本项目生成的是纯静态 HTML 文件，所有资源链接均以相对路径或绝对 URL 形式嵌入页面中，本身不依赖外部 API 或实时数据服务。首次构建完成后，您可以将 `dist/` 目录下的产物拷贝至任意 Web 服务器或本地文件系统直接打开使用。但需要注意，页面中收录的链接本身仍然指向外部站点，访问那些站点时需要对应的网络连通性。若希望保持链接状态信息的新鲜度，建议定期执行 `pnpm run check` 命令以生成最新的可用性报告。

**问：我如何自定义页面顶部的品牌名称、Logo 或配色主题？**

答：品牌相关信息位于 `src/templates/layout.hbs` 中的 `{{brand}}` 占位符，您可以直接修改该模板文件中的静态文本。Logo 图片请替换 `public/logo.svg` 文件，并确保尺寸与现有设计一致。配色主题定义在 `src/styles/base.css` 中的 CSS 自定义属性（`--color-primary`、`--color-background` 等），修改后重新构建即可生效。若需要新增完整的暗色或高对比度主题，建议复制 `theme-dark.css` 并调整其中的变量覆盖值。

**问：我提交的链接审核周期大约是多久？如果链接失效了会如何处理？**

答：维护者通常会在 48 小时内对新增的 Pull Request 进行初步审核，重点检查链接内容是否与分类匹配、站点是否可正常访问、以及是否存在安全风险。对于已收录链接的失效报告，我们会在 Issue 中标记 `expired` 标签，并尝试在 7 个工作日内联系原提交者确认是否更换地址或移除该条目。若无法联系到提交者且链接持续不可用超过 30 天，该条目将从主分支中移除，并在 CHANGELOG 中记录移除原因。
