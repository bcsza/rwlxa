# Tangram Navigation

Tangram Navigation 是一个面向开发者和技术研究人员的综合性技术资源导航与外部链接聚合系统。本项目并非传统意义上的应用软件，而是一个高质量、高可用的在线技术资源目录，旨在解决技术社区中优质外文资料、工具站点与前沿内容分散、难以检索与持续跟踪的问题。项目定位为“技术资源的入口”，通过人工筛选与结构化分类，帮助用户快速定位所需信息，减少无效搜索时间。

项目默认以静态站点形式交付，内容完全基于 Markdown 与 YAML 数据源构建，支持一键部署至各类静态托管服务。其设计理念强调简洁、高速与可控，不依赖后端数据库，不追踪用户行为，适合作为个人或团队内部的技术收藏夹与知识库基础框架。

## 功能概览

- **精选外链分类目录** 按技术领域、资源类型与语言对收录链接进行多维度划分，每一条外链均附带简要说明与标签。

- **全站全文检索** 基于客户端索引实现毫秒级关键词匹配，支持对标题、描述、标签与正文摘要的联合搜索。

- **多级筛选与排序** 用户可按分类、热度、更新时间或字母顺序对资源列表进行动态过滤与重排。

- **自定义资源订阅源** 支持生成 RSS/Atom 格式的输出流，便于用户通过阅读器跟踪新增或更新的资源条目。

- **可编辑的数据格式** 所有资源条目以 YAML 文件存储于仓库目录中，支持批量导入、脚本校验与版本控制。

- **响应式布局与暗色主题** 前端界面适配桌面、平板与移动设备，并跟随系统偏好或用户手动切换明暗主题。

- **轻量级无依赖运行时** 构建产物仅为静态 HTML、CSS 与 JavaScript 文件，无需额外运行时环境或外部 CDN 依赖。

- **开放的数据导出接口** 提供 JSON 格式的完整资源导出端点，便于第三方工具或自动化流程进行二次处理。

## 应用场景

1. **个人技术收藏夹的规范化管理** 开发者可使用本项目替代浏览器自带的散乱书签，将所有常用技术文档、API 参考、在线工具与社区讨论链接集中存放，并通过标签与分类进行体系化管理。

2. **团队内部知识库的入口层** 技术团队可将本项目部署为内部导航页，统一收录公司使用的云平台、监控系统、日志工具、代码仓库与设计规范等关键地址，降低新成员的上手成本。

3. **开源项目文档的补充索引** 开源维护者可在项目的 README 或官网中引用本导航站中对应的分类链接，为用户提供更丰富的上下文参考，例如语言规范、最佳实践示例或性能测试工具。

4. **技术社区的内容聚合试点** 社区运营者可以基于本项目的静态生成能力，快速搭建一个垂直领域的资源合集，例如“云原生工具链大全”或“Rust 学习资源导航”，并通过订阅源保持内容更新。

## 快速开始

以下命令适用于 Unix/Linux 及 macOS 系统，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆仓库至本地
git clone https://github.com/tangram-navigation/tangram.git
cd tangram

# 安装依赖（本项目使用 Node.js 构建工具链）
npm install

# 执行构建流程，生成静态站点至 dist 目录
npm run build

# 启动本地预览服务器（默认端口 8080）
npm run serve
```

执行完毕后，访问 `http://localhost:8080` 即可查看站点。若需部署至生产环境，将 `dist` 目录下的所有文件上传至对象存储或静态托管服务即可。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 用于运行构建脚本与依赖管理，推荐使用 nvm 管理版本 |
| npm | 9.x 或 10.x | 随 Node.js 自动安装，用于安装项目依赖包 |
| Git | 2.30 及以上 | 用于克隆仓库与版本控制，Windows 用户需安装 Git for Windows |

| 静态 Web 服务器 | 任意（如 nginx、Caddy、Vercel） | 生产环境需要提供 HTTP 服务，开发环境使用内置预览服务器 |
| 现代浏览器 | 最近两个主要版本（Chrome、Firefox、Edge、Safari） | 前端界面依赖 ES6+ 与 CSS Grid 特性，不支持 IE 11 |

## 贡献指南

1. **提交资源推荐** 若您希望新增一条外链，请先在 `src/data/links/` 目录下新建一个 YAML 文件，按照 `docs/data-maintenance/` 中的字段规范填写标题、URL、分类、标签与简要描述，然后通过 Pull Request 提交。提交前请运行 `npm run validate` 确保格式正确。

2. **报告问题或改进建议** 使用 GitHub Issues 提交 bug 报告或功能请求。请清晰描述复现步骤、预期行为与实际行为，并附上浏览器版本与操作系统信息。对于样式或交互改进，请尽量附带设计草图或参考链接。

3. **代码贡献流程** 派生本项目至个人账户，创建特性分支（如 `feature/search-optimization`），完成修改后确保所有单元测试通过（`npm run test`），并更新相关文档。提交 Pull Request 时请填写变更摘要，并关联相关 Issue 编号。

4. **翻译与本地化** 若您希望将界面文本或文档翻译为其他语言，请在 `src/assets/i18n/` 目录下新增对应语言文件，并参照已有翻译模板进行填充。翻译完成后提交 Pull Request，维护者将协助合并。

5. **维护者审核标准** 所有 Pull Request 需要至少一名维护者进行代码审查，检查内容包括但不限于代码风格、性能影响、文档同步以及测试覆盖。审核通过后由维护者合并至主分支。

## 常见问题

**Q: 静态站点如何实现搜索功能？**  
A: 本项目在构建阶段会扫描所有资源 YAML 文件，提取标题、描述、标签和分类信息，生成一个轻量级倒排索引 JSON 文件。前端搜索时加载该索引并在浏览器内存中进行匹配，因此无需后端服务即可实现全文检索。索引文件大小通常控制在 200 KB 以内，对加载速度影响极小。

**Q: 如何自定义主题颜色或布局？**  
A: 所有样式变量定义在 `src/assets/css/variables.css` 中，您可以直接修改 CSS 自定义属性（如 `--color-primary`、`--color-background`）来调整主题色。布局模板位于 `src/layouts/` 目录下，基于 Nunjucks 语法，您可以根据需要增删区块。修改后重新运行 `npm run build` 即可生效。

**Q: 数据文件能否从远程 API 动态加载？**  
A: 本项目设计为完全静态，默认从本地 YAML 读取数据。若您需要对接远程 API，可修改 `scripts/build.js` 中的 `fetchData` 函数，将其替换为 `fetch` 调用并处理返回的 JSON 结构。但请注意，这样做会失去完全静态化的优势，且构建过程需要网络访问权限。
