# LinkVault

LinkVault 是一个面向技术内容创作者、开源项目维护者以及个人知识管理实践者的轻量级外链资源整合与导航工具。它并非一个传统的爬虫或书签管理器，而是一个基于静态标记语言生成的、结构清晰的技术资源聚合站。LinkVault 允许用户通过编写简单的配置文件，将散落在互联网各处的技术文档、工具站点、视频教程和社区论坛按照自定义分类进行归集，并生成可供公开访问或私有部署的只读导航页面，从而解决技术资料碎片化、难以体系化分享与回溯的问题。

## 功能概览

- **声明式资源清单管理**：通过 YAML 或 JSON 格式的配置文件定义资源分类、标签和描述，无需数据库，无需后端服务，仅依赖静态文件生成逻辑。
- **多维度资源检索与筛选**：支持按分类标签、来源站点、内容类型（文档、视频、工具、社区）进行实时客户端筛选，并整合关键词模糊匹配，提升海量链接的查找效率。
- **自定义主题与布局**：内置三套基础 UI 主题（浅色、深色、自适应），并允许用户通过 CSS 变量微调色彩与字体，以适应不同品牌或阅读偏好。
- **自动化元数据抓取辅助**：在构建阶段，LinkVault 可尝试抓取目标资源的页面标题、描述与图标，并作为资源卡片的后备展示信息，减少手动输入负担。
- **链接可用性健康检查**：提供可选的定时任务或构建时钩子，对已收录的 URL 进行连通性探测，标记失效链接，并生成健康报告，便于维护者及时清理或更新。
- **静态站点生成与单页应用双模式**：既支持构建完整的静态 HTML 目录树，适用于 GitHub Pages 或任何 Web 服务器；也支持开发环境下以 SPA 形式运行，方便本地预览与调试。
- **OPML 与 JSON 数据导出**：支持将当前资源集合导出为标准 OPML 大纲格式或纯 JSON 结构，便于导入其他 RSS 阅读器、笔记软件或进行二次数据处理。

## 应用场景

- **开源项目文档外链集合**：当一个开源项目需要引用大量外部依赖、学习资料、社区讨论帖或视频讲解时，维护者可以使用 LinkVault 构建一个独立的 `Resources` 页面，放置在项目 Wiki 或侧边栏，帮助贡献者快速理解项目背景，而无需在 README 中堆积冗长的链接列表。
- **技术团队内部知识周报**：技术负责人或架构师可以将团队本周关注的博客文章、新发布的技术框架、性能调优案例通过 LinkVault 汇总，并部署在内网服务器上。团队成员能够通过标签快速过滤前端、后端、运维等不同方向的内容，形成体系化的周报文化。
- **个人学习路线图配套资源索引**：正在系统学习某个技术栈（例如 Rust 语言或 Kubernetes）的个人开发者，可以按照学习阶段（基础、进阶、实战）建立分类，将优质的文档、视频、交互式练习站点一一收录。LinkVault 提供的健康检查功能可定期提醒失效链接，避免学习路径中的“断头路”。
- **技术会议与活动资料归档**：组织线下或线上 meetup 后，主办方可以将演讲 PPT 链接、演示视频回放、相关代码仓库以及提问环节的 FAQ 参考链接通过 LinkVault 统一发布。参会者可以一目了然地获取所有会后资料，且无需登录任何第三方平台。

## 快速开始

以下步骤适用于 macOS / Linux 及 Windows WSL 环境。确保已安装 Node.js 16.x 或更高版本以及 Git。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 npm）
npm install

# 复制示例配置文件并编辑
cp config/resources.example.yml config/resources.yml

# 运行开发服务器，预览默认资源列表
npm run dev
```

执行完上述命令后，在浏览器中访问 `http://localhost:5173` 即可查看 LinkVault 示例界面。如需构建生产环境静态文件，请执行 `npm run build`，生成的 `dist` 目录即为可直接部署的静态站点内容。

## 常见问题

**问：LinkVault 是否必须联网才能工作？**  
答：并非必须。元数据抓取功能默认在开发构建阶段启用，但您可以修改配置文件中的 `fetchMetadata` 选项为 `false` 来彻底关闭网络请求。此时资源卡片将仅显示您在配置文件中手动填写的标题和描述，完全离线可用。

**问：我收录了成百上千个链接，构建速度会不会很慢？**  
答：LinkVault 内部实现了基于文件哈希的缓存机制。对于未变更的配置项，构建时会复用上次抓取的元数据，从而大幅提升增量构建速度。首次构建时，您可以通过 `--parallel` 参数控制并发抓取数量，避免对目标站点造成过大压力。

**问：如何迁移我现有的书签数据（例如从浏览器导出的 HTML）？**  
答：目前官方未提供直接转换工具，但您可以使用社区维护的 `linkvault-importer` 插件（独立仓库），该插件支持将 Netscape 格式书签文件转换为 LinkVault 兼容的 YAML 配置。您也可以编写自定义脚本，利用 `src/utils/` 中的标准化函数将 JSON 数据映射为配置格式。
