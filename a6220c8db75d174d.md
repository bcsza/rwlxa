# PaperLink Navigator

PaperLink Navigator 是一个面向科研工作者与工程技术人员的开源学术资源导航站，专注于收录、整理与分类高质量的学术工具、开放获取期刊、预印本平台及科研辅助站点。项目定位为“轻量级学术外链中继枢纽”，不存储任何实体文件或元数据，仅提供结构化 URL 入口与场景化使用建议，帮助用户从海量分散信息中快速定位可信资源。

项目主要服务于研究生、高校教师、实验室研究员以及独立开发者，解决科研流程中常见的三方面问题：文献获取渠道碎片化、不同领域工具发现成本高、以及跨境访问场景下的备用站点遗忘。通过人工维护的精选列表与清晰的应用场景映射，本项目显著降低科研信息检索的重复劳动，提升文献调研与实验数据处理的整体效率。

## 功能概览

- **主题分类导航** 按学科领域与使用阶段划分资源类别，涵盖文献检索、数据分析、写作辅助、期刊投稿等方向。

- **多来源交叉验证** 针对同一资源提供多个镜像或替代站点入口，帮助用户规避单点访问失败或网络限制。

- **可用性状态标记** 在资源列表中标注近期可达性状态，区分稳定主站、备用站与临时维护节点。

- **轻量级本地部署** 项目纯静态化构建，无需后端服务与数据库支持，可克隆至本地直接打开 index 文件使用。

- **快速关键词检索** 内置前端全文模糊匹配功能，支持按资源名称、标签、描述进行即时筛选。

- **自定义扩展接口** 提供 JSON 格式的资源数据文件，用户可自行增删条目或导入个人书签集合。

- **浏览器跨设备同步** 配合浏览器书签或自建 WebDAV，可实现导航配置在多终端间的一致性维护。

## 应用场景

- **文献全文获取** 用户在校外或无机构订阅权限时，可通过导航中收录的开放获取镜像站与预印本平台查找论文全文，减少付费墙阻碍。

- **跨语言写作辅助** 科研人员在撰写英文论文或项目申报书时，利用导航内的语法检查工具、术语库与模板仓库，提升非母语写作质量。

- **实验数据可视化** 工科与数据科学方向用户快速跳转至在线图表绘制、统计检验与流式数据处理工具，避免在本地重复安装大型软件。

- **期刊信息核查** 投稿前通过导航中的期刊黑名单查询、影响因子跟踪与审稿周期统计站点，辅助选择合规且高效的发表渠道。

- **离线应急访问** 在网络环境受限或主站故障时，通过导航记录的备用域名与 IP 直连入口维持科研工作流的连续性。

## 快速开始

```bash
# 克隆仓库至本地
git clone https://github.com/paperlink-navigator/paperlink-navigator.git

# 进入项目根目录
cd paperlink-navigator

# 安装依赖（本步骤仅在需要二次开发时执行，纯浏览用户可跳过）
npm install

# 本地开发预览（启动后访问 http://localhost:8080）
npm run dev

# 生产环境构建（输出静态文件至 dist 目录）
npm run build
```

## 贡献指南

1. 在 GitHub 仓库页面点击 Fork 按钮创建个人复刻，随后将复刻仓库克隆至本地开发环境，确保使用最新的 main 分支作为基线。

2. 新建以 feature/ 或 fix/ 为前缀的分支，遵循语义化命名规范，例如 feature/update-mirror-list 或 fix/search-case-sensitive，避免直接在主分支上修改。

3. 针对资源数据的增删改操作，统一编辑 src/data/resources.json 文件，同时更新对应分类的 categories.json 以维持映射一致性，所有变更需附带注释说明添加理由或失效依据。

4. 提交变更前运行 npm run test 与 npm run lint 确保所有单元测试通过且无代码风格告警，提交信息采用 Conventional Commits 格式，例如 chore: remove outdated gateway entry。

5. 向原仓库发起 Pull Request，描述中清晰列出变更内容、影响范围以及至少一个实际使用示例，等待维护者审核与合并，合并后即自动触发构建部署流程。

## 常见问题

**问：导航站中的某些链接无法访问，我应该如何处理？**

答：首先尝试刷新页面或更换网络环境（例如切换至手机热点），排除本地网络波动。若仍无法打开，请通过 GitHub Issues 提交问题报告，附上链接地址、错误提示截图以及您的地理位置大致信息。维护团队会在 48 小时内复核并标记失效状态，同时寻找替代入口或更新为最新可用地址。

**问：我能否将本项目部署在完全离线内网环境中？**

答：可以。由于项目不调用任何外部 API 且资源数据全部本地化存储，您只需将 dist 目录复制到内网 Web 服务器（如 Nginx、Apache 或 Python http.server）即可。注意内网环境若需访问外部链接，需保证终端设备具备相应的网关与 DNS 解析权限，导航站本身不提供代理或穿透功能。

**问：如何批量导入我自己的书签集合，而不逐条手动录入？**

答：项目支持通过 src/data/importers 目录下的辅助脚本将浏览器导出的 HTML 书签文件或 CSV 表格转换为符合 resources.json 格式的数据。具体操作请参考 docs/developer-api.md 中的“批量导入”章节，该脚本会进行自动去重与字段映射，执行前建议先备份原始数据文件。
