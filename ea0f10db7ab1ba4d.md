# HexaLink Hub

HexaLink Hub 是一个面向技术内容创作者、开源项目维护者以及互联网资源管理者的外链聚合与导航系统。该项目并非传统的网址导航站，而是一套基于静态站点生成逻辑的资源映射框架，用于解决多源、多类型外部链接的归类、版本追踪与可读性呈现问题。目标用户包括技术文档撰写者、社区运营人员以及需要维护大量外联引用的开发团队。通过结构化的数据组织方式，HexaLink Hub 能够将散落的第三方资源转化为清晰、可维护的引用网络，有效降低链接漂移和资源失效带来的维护成本。

## 功能概览

- **分级资源映射**：支持按内容主题、来源域名或业务场景对 URL 进行多级分类，并生成对应的浏览索引。
- **链接状态标记**：内置链接可访问性检查接口，可对资源列表中的每个条目附加状态标签（有效、失效、临时重定向）。
- **静态快照生成**：基于配置文件自动生成纯静态 HTML 目录页面，便于部署至任何 Web 服务器或对象存储。
- **批量导入与校验**：支持从 CSV 或 JSON 批量导入链接清单，并自动校验 URL 格式合规性与协议一致性。
- **版本化引用记录**：每次更新资源列表时自动生成变更摘要，记录新增、移除或修改的链接，便于回溯。
- **自定义元数据扩展**：每条资源条目允许附加备注、标签、更新周期、维护人字段，适配企业级知识库需求。
- **搜索与过滤接口**：提供轻量级前端搜索能力，支持按关键字、域名后缀或分类快速定位目标资源。

## 应用场景

- **开源项目文档站外链管理**：技术文档中常需引用外部规范、依赖仓库或参考文章，HexaLink Hub 可集中维护这些引用，确保文档构建时引用来源一致且可审查。
- **社区资源周刊自动化生成**：技术社区运营人员可将每周推荐的阅读列表、工具站点或视频教程录入系统，自动生成带分类的周报页面。
- **多环境配置的外部依赖映射**：企业内部多个开发环境（测试、预发布、生产）可能依赖不同的外部 API 地址或数据源，利用本系统可按环境标签分别维护并导出对应配置。
- **合规审计中的外部引用溯源**：对于需要满足出口管制或数据合规要求的项目，可借助本系统记录每个外部资源的引入时间、用途和审批状态，简化审计材料整理。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，要求已安装 Git 与 Node.js 18+。

```bash
# 克隆项目仓库
git clone https://github.com/hexalink-hub/hexalink-hub.git
cd hexalink-hub

# 安装依赖
npm install

# 构建资源索引并启动本地预览服务
npm run build
npm start
```

执行完成后，访问控制台输出的本地地址（默认为 http://localhost:3000）即可查看资源导航界面。如需自定义资源列表，请编辑 `data/sources.json` 文件并重新执行 `npm run build`。

## 贡献指南

1. 首先在 GitHub 上 fork 本仓库，并将个人分支克隆至本地。建议使用 `dev/` 前缀命名特性分支，例如 `dev/add-streaming-category`。
2. 修改或新增资源条目时，请严格按照 `data/sources.json` 中既有的字段结构操作，确保每个条目包含 `id`、`url`、`category`、`remark` 四个必要字段。提交前运行 `npm run validate` 校验数据完整性。
3. 若新增分类或调整页面布局，需同步更新 `docs/user-guide/` 下对应的操作说明。文档变更须与代码变更在同一 PR 中提交。
4. 提交信息请遵循 Conventional Commits 规范，使用 `feat:`、`fix:`、`docs:`、`chore:` 等类型前缀。PR 描述中需说明变更动机、测试结果以及是否影响现有构建流程。
5. 合并前需通过 GitHub Actions 中的自动化测试（包含链接格式校验与构建成功测试），并至少获得一位项目维护者的 approval。

## 常见问题

**Q：如果外部链接失效或域名变更，系统如何处理？**

A：系统本身不会自动修改用户配置的 URL。但 `npm run check` 命令会生成一份失效链接报告，列出所有返回 4xx 或 5xx 状态码的条目。建议用户定期执行该命令，并根据报告手动更新 `sources.json`。对于频繁变动的资源，可在元数据中标记 `check_interval` 字段以调整检查频率。

**Q：生产环境下如何部署生成的静态页面？**

A：执行 `npm run build` 后，所有静态文件会输出至 `static/` 目录。您可以将该目录整体上传至 Nginx、Apache、S3 或 Cloudflare Pages 等任何支持静态托管的服务。无需额外后端服务，所有搜索和过滤功能均在前端完成。

**Q：能否直接使用 `sources.json` 中的数据而不生成 HTML？**

A：可以。`sources.json` 是标准的 JSON 文件，可被任何编程语言读取。您可以通过 `npm run export -- --format=json` 或 `--format=csv` 将当前数据导出为其他格式，以便集成至您的其他工具链中。
