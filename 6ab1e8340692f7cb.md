# FilmNav

FilmNav 是一个面向影视行业从业者与内容研究人员的专业导航与资源聚合项目。该项目并非一个传统的在线播放或下载平台，而是一个精心编排的技术索引目录，旨在解决影视制作、发行与研究过程中信息源分散、技术文档查找困难以及行业标准更新滞后等问题。目标用户包括独立制片人、影视技术监制、数字发行经理、影视院校师生以及影音发烧友。

项目通过对互联网上散落的影视相关技术资源、行业数据接口以及专业工具站点进行逻辑分类与状态监控，帮助用户快速定位到权威的编码标准、设备参数库、发行渠道规范及市场分析原始数据。FilmNav 定位为影视工业流程中的信息中继站，所有外链均在新窗口以原始形态呈现，确保用户直接触达目标资源，避免中间层的信息污染或访问失效。

## 功能概览

**影视发行渠道索引** 提供主流数字发行与院线对接平台的快速访问入口，覆盖院线排片、数字拷贝分发及密钥管理的基础服务链路。

**设备与拍摄参数库** 聚合摄影机、录音设备及后期制作硬件的官方技术规格表与固件更新日志，便于制作团队在前期筹备时进行跨设备数据比对。

**行业标准与规范文档** 整理国内外影视制作、广播传输及流媒体编码的相关标准文档发布页，包括色域、音频响度及字幕格式的现行规范。

**市场数据与票房统计** 链接至第三方权威票房数据统计源及行业分析报告原始发布页面，为项目立项与市场调研提供真实数据支撑。

**后期制作工具链导航** 归类色彩校正、视觉特效合成及声音后期处理等专业软件的官方文档库与用户社区入口。

**线下活动与展映信息** 收录电影节报名系统、技术研讨会注册页及行业展会信息发布站，方便从业者获取线下交流机会。

**内容分发网络状态查询** 提供视频流媒体分发相关的网络状态检测工具及服务商状态仪表板入口。

## 应用场景

影视制作公司的技术监制在筹备新项目时，可通过本导航快速查阅不同数字摄影机的官方动态范围与色彩采样参数文档，同时对比多家发行渠道的交付规格要求，从而在开机前制定统一的技术执行标准。

独立纪录片导演在后期制作阶段需要查询特定编码格式的封装参数或寻找开源的母版制作工具，本导航中收录的后期工具链与规范文档页面能够提供直接的官方说明链接，减少因搜索错误信息导致的兼容性问题。

影视院校的研究生在撰写关于当代数字发行模式的论文时，需要引述权威的票房数据与发行窗口期统计。本导航中的市场数据板块提供了直达原始统计页面的路径，确保研究数据的可溯源性。

发行公司的项目经理在对接海外平台时，需要定期核对不同地区的内容分级标准与字幕交付格式。通过本导航的行业标准模块，可快速定位相关组织的官方规范发布页，提高跨部门协作效率。

## 快速开始

以下指令适用于 Linux 及 macOS 环境，Windows 用户请使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/filmnav/filmnav-index.git

# 进入项目根目录
cd filmnav-index

# 安装依赖（项目基于 Node.js 环境，需使用 npm）
npm install

# 启动本地开发服务器，默认监听端口 3000
npm run dev
```

## 贡献指南

贡献者需先阅读项目行为准则并在拉取请求中遵守相关约定。所有贡献均视为同意将代码或数据以 MIT 许可证进行授权。

1. 复刻项目仓库至个人账户，并在本地克隆该复刻版本。请确保本地 Git 配置了正确的用户名与邮箱，以便提交记录可被追溯。

2. 创建新的功能分支，分支命名应遵循 `feat/` 或 `fix/` 前缀加简要描述，例如 `feat/add-documentary-category`。禁止在主分支上进行直接修改。

3. 若需新增或修改导航链接条目，请编辑 `config/navigation.json` 文件，并严格按照 `docs/data-format-spec.md` 中定义的格式填写字段。提交前请运行 `npm run validate` 校验数据合法性。

4. 完成修改后，发起合并请求至主仓库的 `main` 分支。请求描述中应详细列出变更内容、测试结果以及是否影响现有功能。核心维护者将在三个工作日内进行审查。

## 常见问题

**问：项目中的链接有时无法访问，是否会影响本站的正常使用？**

答：本站仅作为导航索引，不代理或缓存任何外部资源内容。链接的可用性取决于目标服务器自身的状态。本站会在本地存储中记录上次访问时的状态，但建议用户在使用时自行确认链接的有效性。若发现大量失效链接，欢迎通过贡献指南提交更新请求。

**问：如何请求添加新的分类或删除现有分类下的条目？**

答：所有分类结构及条目的增删改均通过 `config/navigation.json` 文件管理。请按照贡献指南的流程提交修改请求。在发起合并请求前，请确保已充分说明新增分类的定位或删除条目的具体理由，以便维护者评估改动对现有用户的影响。

**问：项目是否支持自定义样式或布局切换？**

答：当前版本支持通过 `config/theme.json` 修改基础色彩变量。如需更深层次的布局调整，建议在本地复刻项目后自行修改 `src/components` 中的组件逻辑。官方维护团队暂不提供多样式主题的在线切换功能，但欢迎社区贡献相关实现。
