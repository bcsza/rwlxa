# LinkVault

LinkVault 是一个面向技术内容创作者与资源整理者的开源外链资产管理系统。它并非传统意义上的网址导航站，而是一套围绕特定垂直领域（如影视技术、流媒体工具、设计素材）进行高密度外链资源整合与结构化展示的静态站点生成框架。项目定位服务于个人知识库维护者、小型技术社区运营者以及需要批量管理外部参考链接的研发团队，帮助其在不依赖数据库的前提下，通过标记语言与简单的目录约定，实现数千级外链资源的分类、版本控制与快速发布。

LinkVault 不提供爬虫或自动采集功能，强调人工筛选与语义化标注，确保每个收录的 URL 都经过上下文校验。项目内置链接状态检测脚本，可定期输出失效链接报告，解决外链汇总类项目最常见的内容腐烂问题。

## 功能概览

**分级目录导航体系** 基于业务领域与资源类型构建三层索引结构，支持无限级子分类嵌套，每级分类均可独立生成聚合页。

**富元数据标注模板** 为每条外链提供标题、描述、标签、收录日期、维护人、关联项目共六个维度的扩展字段，便于后续检索与审计。

**链接健康度监控** 集成异步 HTTP 状态检查器，按可配置周期对全部资源执行 HEAD 请求，生成可视化健康度看板与 CSV 格式异常报告。

**静态站点一键生成** 内置基于 Python 的渲染引擎，可将 YAML 格式的资源索引编译为响应式 HTML 页面，输出产物全量静态，适合托管于任何对象存储或 CDN。

**多格式数据导入导出** 支持 JSON、CSV、Markdown 表格三种数据交换格式，便于与外部系统（如 Notion、Airtable、飞书多维表格）进行批量同步。

**私有化部署友好的权限占位** 预留基于环境变量的只读/可写双模式开关，方便团队内部将同一份源码同时用于公开导航页与私有资产台账。

**变更审计日志** 自动记录每次资源增删改的操作人、时间戳与变更摘要，日志以结构化文件存储于独立目录，支持回溯至任意历史版本。

## 应用场景

技术博客作者的内容支撑库 博主在撰写系列教程时，可将所有引用的官方文档、工具站、参考示例统一收录于 LinkVault，并在每篇文章末尾通过嵌入片段动态展示相关资源列表，避免手动维护文末链接的重复劳动。

小型设计团队的内部素材雷达 团队可将常用的字体下载站、无版权图库、配色工具、图标集按项目维度分类存放，新成员入职时通过浏览 LinkVault 生成的导航页即可快速了解团队常用的外部服务生态。

开源项目文档站的外部依赖索引 开源项目维护者可在项目 docs 目录下集成 LinkVault，集中管理所有依赖库主页、API 参考、社区论坛、镜像源地址，随项目版本一同发布，确保文档中的外部链接始终与版本基线对应。

线下技术沙龙的活动资源归档 活动组织者可将每期分享嘉宾提到的工具、论文链接、视频回放地址按场次归档，生成独立的会务资源页供参会者事后查阅，取代零散的聊天记录粘贴。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Python 3.9 及以上版本。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装项目依赖（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 请使用 venv\Scripts\activate
pip install -r requirements.txt

# 执行示例资源索引的构建与预览
python build.py --source data/sample.yaml --output dist/
python -m http.server -d dist  # 默认监听 8000 端口
```

## 常见问题

Q: 资源索引文件支持哪些字段，是否可以自定义扩展字段？

A: 默认每个条目支持 title、url、description、tags、maintainer、related 共六个字段。其中 tags 为字符串列表，related 为 URL 字符串，用于关联内部或外部参考链接。若需扩展额外字段，可在 data/ 目录下任意 YAML 文件中直接添加新键值对，构建器会将未知字段原样透传至模板上下文，模板中可通过 `entry.custom_field` 方式调用，无需修改核心解析代码。

Q: 链接健康度检查是否会影响生产环境的访问日志？

A: 检查器使用 requests 库的 Session 对象，默认 User-Agent 为 `LinkVault-HealthCheck/1.0`，且仅发送 HEAD 请求（若服务器支持），不下载响应体。检查间隔默认设置为 7 天，并通过随机抖动（±12 小时）分散请求时间，避免对目标服务器造成瞬时压力。用户可在 config.yaml 中调整 `checker.interval_days` 与 `checker.timeout_seconds` 参数。

Q: 如何将现有书签（如 Chrome 导出的 HTML 书签）批量导入 LinkVault？

A: 项目暂未提供直接解析 Chrome 书签格式的转换器，但支持 CSV 导入。用户可先使用浏览器书签管理器将书签导出为 HTML，再通过在线工具或 Python 脚本（如 BeautifulSoup）将其转换为包含 `title,url,tags` 三列的 CSV 文件，最后利用 LinkVault 自带的 `import-csv` 子命令（参见 `python build.py import-csv --help`）完成批量入库。
