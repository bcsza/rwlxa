# OpenLinkHub

OpenLinkHub 是一个面向技术社区与网络资源研究者的高质量外链聚合导航项目。本项目并非简单的书签收藏，而是通过人工筛选与结构化编排，构建出一个覆盖影视、工具、学术、开发等多维度的可维护资源索引系统。项目定位为“技术驱动的网络资源中转站”，旨在解决个人收藏夹杂乱、链接失效快、跨领域资源发现困难等常见痛点，适用于个人开发者、技术博主、研究助理以及任何对高效信息管理有需求的用户。

## 功能概览

- **多级分类索引体系**：按内容主题与使用频次划分资源类别，支持快速定位特定领域的高价值链接，减少无效检索时间。
- **链接存活状态标记**：内置周期性检查机制，对收录的每一个外链进行可用性探测，并在索引中标注异常状态，确保资源列表的实用性与时效性。
- **轻量级本地搜索**：提供基于关键词的模糊搜索功能，允许用户在当前资源集合中快速筛选匹配项，无需依赖外部搜索引擎。
- **简洁只读展示模式**：面向查阅场景优化视觉密度，去除冗余动画与交互干扰，使信息层级清晰，适合长时间阅读与比对。
- **可扩展数据格式**：所有链接数据以结构化的 YAML 或 JSON 文件存储，用户可直接编辑或通过脚本批量导入导出，便于与其他工具链集成。
- **离线可用快照**：支持生成当前资源列表的静态 HTML 快照，便于在内网环境或离线状态下浏览核心索引内容。
- **自定义标签系统**：允许用户为每个链接添加自定义标签（如“需代理”“高频”“备用”），实现个人化的分类逻辑，同时不影响公共索引结构。

## 应用场景

- **技术调研与竞品分析**：开发者在启动新项目前，可通过 OpenLinkHub 快速触达多个垂直领域的资源站点，例如影视技术栈、流媒体工具、前端框架社区等，大幅缩短信息收集周期。
- **学术资料与媒体研究**：研究人员可利用本项目的分类索引，系统性地收集特定主题下的网络资源，例如影视数据源、字幕组历史存档、流媒体协议文档等，便于构建研究参考库。
- **个人知识库基础骨架**：知识管理爱好者可将 OpenLinkHub 作为其个人知识库的“互联网入口”模块，与本地笔记工具（如 Obsidian、Logseq）结合，形成内外联动的知识网络。
- **团队共享书签替代方案**：小型团队或兴趣小组可使用本项目作为内部共享资源池，替代浏览器自带的共享书签功能，获得更清晰的分类、更稳定的链接管理和更透明的状态反馈。

## 快速开始

以下命令将帮助你在一台装有 Git 和 Node.js 的机器上快速启动 OpenLinkHub 的本地实例。

```bash
# 克隆项目仓库
git clone https://github.com/openlinkhub/openlinkhub.git

# 进入项目目录
cd openlinkhub

# 安装依赖（使用 npm 或 yarn）
npm install

# 启动开发服务器，默认访问 http://localhost:3000
npm run start
```

## 贡献指南

我们欢迎并鼓励社区贡献，无论是新增有价值的链接、修复文档错误，还是改进核心功能。请遵循以下步骤参与贡献：

1. 在 GitHub 上 Fork 本仓库，并在本地克隆你的 Fork 副本。请确保你的分支基于最新的 main 分支。
2. 对于链接新增或修改，请编辑 `data/links/` 下对应的 YAML 文件，并严格遵守文件中定义的字段格式与缩进规则。对于代码变更，请确保新增或修改的代码包含必要的单元测试，且所有现有测试用例通过。
3. 提交前，请运行 `npm run lint` 和 `npm run test` 进行代码规范检查与测试，并更新相关文档（如 user-guide.md）以反映你的变更。
4. 提交一个清晰的 Pull Request，描述变更内容、动机以及任何可能的影响范围。PR 描述中请引用相关 Issue（如有）。
5. 等待维护者审核。我们会在 3 个工作日内给予反馈，可能要求调整或补充信息。合并后，你的贡献将出现在下一版本的资源索引中。

## 常见问题

问：链接存活状态检查是如何工作的？是否会频繁请求外部站点？

答：项目内置的检查器默认采用异步并发请求，并设置了 5 秒超时限制。检查频率由 `config/settings.json` 中的 `checkInterval` 字段控制，默认为 24 小时一次，仅检查状态码（HEAD 请求），不会下载页面内容，对目标站点影响极小。用户也可通过命令行工具 `npm run check` 手动触发检查。

问：我是否可以完全离线使用 OpenLinkHub？

答：可以。你只需在联网状态下执行一次 `npm run build` 生成静态站点文件，所有资源数据与索引信息都会被嵌入到生成的 HTML 中。此后，你可以将 `dist/` 目录下的内容拷贝到任何离线环境，直接通过浏览器打开 `index.html` 即可查阅完整的资源列表，但搜索功能将降级为仅匹配已生成的索引数据，无法动态筛选。

问：如何备份我自己添加的标签和注释？

答：所有自定义标签与个人注释都存储在 `data/tags/custom-tags.json` 文件中。该文件独立于公共链接数据，你只需定期备份此文件即可。在升级项目版本时，请注意检查该文件格式是否有破坏性变更，通常我们会保留向后兼容性。
