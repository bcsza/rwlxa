# LinkVault

LinkVault 是一个面向技术内容创作者、开源社区运营者和数字资源管理者的轻量级外链资源聚合与导航系统。该项目旨在解决个人或团队在维护大量外部项目链接、文档站点、工具平台时面临的分散管理、检索效率低下以及链接有效性跟踪困难等问题。LinkVault 不存储任何第三方内容，仅作为结构化外链索引层，提供基于标签、分类和全文检索的快速查找能力，并内置链接健康检查与访问统计模块，帮助用户构建清晰、可维护的私有或公开外链资源库。

LinkVault 定位于中小型技术团队、开源项目维护者以及个人开发者。用户可通过 LinkVault 快速搭建属于自己的技术资源门户，将日常高频访问的文档站、API 参考、设计素材、视频教程等外部链接统一归档，并通过分类树和搜索功能在数秒内定位目标资源。系统采用静态站点生成与动态缓存相结合的策略，既保证前端访问速度，又为管理员提供实时数据更新能力。

## 功能概览

- **多级分类树管理**：支持无限层级目录结构，用户可自定义分类主题，如“视频站点”、“文档参考”、“工具集”等，便于按领域或用途组织链接资源。

- **全文检索与标签过滤**：基于倒排索引实现对链接标题、描述、标签及备注内容的快速检索，同时支持多标签组合筛选，提升海量链接中的查找精度。

- **链接健康状态监控**：后台定时任务自动检测所有外链的可访问性，标记失效链接并记录响应时间，管理员可收到异常汇总通知，确保资源列表的可用性。

- **访问统计与点击分析**：记录每个外链的点击次数、来源页面和访问时间，提供趋势图表，帮助用户识别高频资源并优化导航布局。

- **批量导入与导出**：支持通过 CSV、JSON 和 OPML 格式批量导入外部链接列表，也可将当前资源库导出为通用格式，便于迁移或备份。

- **用户权限与私有链接**：支持多用户协作场景，可设置公开链接与私有链接，私有链接仅对授权用户可见，适合团队内部共享敏感资源。

- **响应式前端主题**：内置深色/浅色两套主题，适配桌面与移动设备，前端页面完全静态生成，保证低延迟访问体验。

## 应用场景

- **技术团队内部文档导航**：研发团队可使用 LinkVault 汇总内部 API 文档、设计稿仓库、CI/CD 流水线地址、日志平台入口等，替代浏览器书签的零散管理方式，新成员入职时可一键获取所有必需工具链接。

- **开源项目外部资源站**：开源项目维护者可利用 LinkVault 搭建项目生态导航页，集中展示社区博客、视频教程、相关工具、镜像站点、第三方依赖文档等，提升社区用户的信息获取效率。

- **个人知识库外链索引**：知识管理爱好者可将日常阅读的专栏文章、在线课程、技术规范、论文预印本等链接分类存档，配合全文检索快速回溯过往参考资料，避免重复查找。

- **内容运营素材库管理**：视频剪辑师、自媒体运营者可整理素材来源站点、音效库、字体下载页、视频模板平台等外链，按项目维度分类，结合访问统计评估各渠道的利用率。

- **教育机构课程资源聚合**：教师或培训组织者可汇总课程所需的在线实验环境、电子教材、习题系统、视频公开课等链接，分学期或分科目建立独立资源目录，供学生自助访问。

## 快速开始

以下步骤适用于 Linux / macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 克隆仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 Python 3.10+ 和 pip）
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并运行开发服务器
python manage.py migrate
python manage.py loaddata fixtures/initial_categories.json
python manage.py runserver --host=0.0.0.0 --port=8080
```

访问 http://localhost:8080 即可进入 LinkVault 管理面板。首次登录请使用默认管理员账号 admin / admin123，并立即修改密码。

## 常见问题

**问：LinkVault 是否可以部署在无公网 IP 的内网环境中？**  
可以。LinkVault 不依赖任何外部云服务（除可选的 Redis 外），所有功能均可完全离线运行。链接健康检查模块会从部署机器所在网络发起请求，若内网环境无法访问公网外链，可通过配置跳过外部链接检测或设置代理。前端页面完全静态生成，无需 CDN 支持。

**问：导入大量链接（超过 10 万条）时，系统性能如何优化？**  
建议在导入前将数据库切换为 PostgreSQL，并关闭链接健康检查的实时触发，改为使用 Celery 异步批量处理。同时，可调整全文索引的更新策略为手动触发而非自动增量，以减轻写入压力。生产环境可启用 Redis 缓存分类树结果，并将搜索请求限流。具体调优参数请参考 docs/deployment.md 中的性能章节。

**问：私有链接与公开链接的权限隔离如何实现？**  
LinkVault 基于 Django 的权限系统，为每个链接资源设置 visibility 字段（public / private）。私有链接仅对创建者及具有 view_private_link 权限的用户可见。管理员可在后台为用户组分配权限，也可在分类级别设置默认可见性。所有 API 接口均会校验当前请求用户的身份和权限，前端则根据用户登录态动态渲染菜单。
