# LinkHub

LinkHub 是一个面向开发人员与技术内容消费者的高质量网络资源聚合与导航工具。项目定位于为技术社区提供一个轻量级、可自部署的入口站点，用于统一管理和快速访问互联网中分散的优质媒体、影视及娱乐内容相关网络资产。LinkHub 本身不存储或分发任何受版权保护的内容，仅以导航与信息聚合的方式降低用户在多个垂直领域站点之间的检索与切换成本，尤其适用于需要频繁查阅不同来源视频、影音或流媒体资源的技术评估人员、内容分析工程师及自动化测试脚本维护者。

## 功能概览

- 智能分类导航 基于资源类型、内容主题与运营主体对收录的网络链接进行自动与手动双重维度分类，支持快速筛选与模糊检索。

- 一键直达式跳转 所有收录链接以原始域名形式直接呈现，并保留协议与子域信息，确保跳转路径与用户预期一致，避免中间页或重定向干扰。

- 本地化配置存储 用户对导航顺序、隐藏或高亮特定条目的偏好全部保存在浏览器本地存储中，无需服务端数据库支持，降低部署与运维成本。

- 站点可用性自检 内置前端定时探测机制，对收录域名进行 HTTP 状态码与响应时间检测，并在界面中以颜色标记异常站点，辅助运维人员快速排查。

- 批量导入与导出 支持以 JSON 或纯文本行格式批量导入外部链接列表，亦可一键导出当前配置与收藏集，便于团队内部共享导航配置。

- 响应式布局与暗色主题 适配桌面端与移动端主流浏览器分辨率，并内置亮色/暗色两套视觉方案，适应不同开发环境下的阅读习惯。

- 轻量级搜索过滤 支持基于域名关键词、分类标签与自定义备注的实时搜索，结果随输入动态更新，提升大规模链接集合下的定位效率。

- 免登录使用 无需注册或登录账户，所有功能在首次加载后即完全可用，降低使用门槛并保护用户隐私。

## 应用场景

- 技术调研与竞品分析 开发团队在对多个视频流媒体平台进行技术栈、页面性能或内容分发策略调研时，可通过 LinkHub 快速切换至不同站点，并结合内置的可用性检测初步评估各服务稳定性。

- 自动化测试环境准备 质量保障工程师在编写端到端测试脚本或内容抓取规则时，需要频繁访问特定测试域名列表，LinkHub 提供集中化入口与批量导出功能，可加速测试环境配置与变更管理。

- 内容运营与编辑工作流 内容运营人员需要每日浏览多个影视资讯或素材站点以获取选题灵感，LinkHub 的分类导航与搜索功能可显著减少重复输入与书签管理混乱的问题。

- 个人知识库与书签增强 开发者可将个人收藏的各类技术文档、演示站点或在线工具链接一并纳入 LinkHub 管理，利用备注与标签体系构建个性化的轻量级知识导航系统。

## 快速开始

以下步骤适用于 Linux / macOS / Windows 操作系统，需预先安装 Git 与 Node.js 16.x 及以上版本。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkhub-io/linkhub.git
cd linkhub

# 安装项目依赖
npm install

# 启动开发服务器，默认监听 http://localhost:3000
npm start
```

## 贡献指南

1. 阅读文档 在提交任何代码或文档变更前，请仔细阅读 docs/ 目录下的开发指南与配置手册，确保理解项目整体架构与编码规范。

2. 报告问题 若发现缺陷或有功能建议，请在 GitHub Issues 中新建条目，并附上清晰的问题描述、复现步骤及运行环境信息（浏览器版本、Node.js 版本、操作系统）。

3. 创建分支 从 main 分支签出新的功能或修复分支，命名格式为 feature/简短描述 或 fix/问题编号-简短描述，避免直接在主干分支上修改。

4. 提交代码 确保所有新增或修改的代码均已通过 ESLint 检查与单元测试，并补充相应的测试用例或文档说明。提交信息应遵循 Conventional Commits 规范。

5. 发起合并请求 将分支推送至远程仓库后，在 GitHub 上发起 Pull Request，并填写变更摘要、测试结果及影响范围。至少需要一名项目维护者审核通过后方可合并。

## 常见问题

问：LinkHub 是否存储或缓存任何第三方站点的内容？

答：否。LinkHub 仅存储域名链接及其分类元数据，不发起对第三方资源的主动抓取、存储或代理转发。所有跳转行为由用户主动点击触发，内容分发与法律责任归属原始站点。

问：项目依赖的外部 CDN 资源离线时是否影响核心功能？

答：核心导航与搜索功能均可在本地缓存生效后完全离线运行。仅首次加载时需从 CDN 获取 UI 框架与图标库，后续访问会优先使用浏览器缓存。若需完全离线环境，可使用 docs/deployment.md 中描述的自托管资源方案。

问：如何批量更新或替换已收录的链接列表？

答：可通过界面右上角的设置面板进入导入/导出功能，选择导出当前配置为 JSON 文件后手动编辑，再通过导入功能覆盖现有列表。亦可通过编辑 src/store/initialLinks.js 文件中的默认数据数组，重启服务后生效。
