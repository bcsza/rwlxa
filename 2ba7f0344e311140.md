# Cinemeta Index

Cinemeta Index 是一个面向数字影音内容研究者、媒体档案管理员与影视资源索引爱好者的轻量级元数据导航系统。本项目不存储、不托管、不转发任何受版权保护的媒体文件，仅以目录形式收集并整理互联网公开的影视信息站点与资源入口，帮助使用者快速定位可用数据源，降低信息检索的时间成本。Cinemeta Index 适用于个人学习、行业调研、内容归档辅助等合法场景，不对任何外部站点的可用性、内容合法性或访问稳定性作任何明示或暗示的担保。

## 功能概览

**公开资源目录聚合**：自动整理并分类展示多个影视信息站点与资源导航入口，提供统一的访问视图。

**可用性轮询示意**：内置简易的 HTTP 状态码检查脚本，可对收录站点进行定期可达性探测并以日志形式输出。

**分类标签与检索**：每个收录条目均可赋予自定义分类标签，支持按名称、标签或站点描述进行快速模糊检索。

**自定义列表导入导出**：允许用户以 JSON 或 CSV 格式导入自己的站点列表，亦可导出当前全量目录作为备份或共享。

**轻量级容器化部署**：提供 Dockerfile 与 Compose 示例，支持在 Linux 服务器、树莓派或本地开发环境中一键启动。

**只读镜像模式**：支持配置为只读镜像站，禁止任何写入操作，适用于公共展示或内网分发场景。

## 应用场景

**影视数据调研与行业分析**：研究人员或媒体分析师可通过 Cinemeta Index 快速访问多个影视信息入口，收集公开的影片基础信息、发行时间、地区分布等元数据，用于制作行业报告或数据分析模型，无需逐一记忆或搜索分散的网址。

**个人媒体库辅助构建**：个人影音爱好者或家庭媒体服务器运维者可将本系统作为导航起点，快速跳转至外部站点获取影片封面、剧情简介、演职员表等补充信息，丰富本地媒体库的元数据字段。

**教育资源与课堂演示**：传媒专业教师或学生可在课堂演示中利用 Cinemeta Index 展示不同地区影视信息站点的页面结构与信息组织方式，作为数字媒体课程的教学辅助工具，帮助学生理解信息架构的多样性。

**内网镜像与离线环境备用**：企业或机构内部网络可部署 Cinemeta Index 作为内部导航首页，收录经过审核的外部站点链接，供内网用户统一访问，同时配合可用性检查功能及时发现失效链接。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，假设已安装 Git、Node.js 与 npm。

```bash
# 克隆仓库
git clone https://github.com/cinemeta/index.git
cd cinemeta-index

# 安装依赖
npm install

# 启动开发服务器（默认监听 3000 端口）
npm run start
```

访问 http://localhost:3000 即可看到首页目录列表。如需构建生产环境静态文件，请执行 `npm run build`，输出目录为 `./dist`。

## 贡献指南

我们欢迎任何形式的贡献，包括但不限于新增站点收录建议、代码缺陷修复、文档改进和功能提案。请遵循以下步骤：

1. 在 GitHub 上 fork 本仓库到您的个人账号，然后 clone 到本地开发环境。请确保您的分支基于最新的 main 分支创建，分支命名建议使用 `feature/描述` 或 `fix/描述` 格式。

2. 完成代码或文档修改后，请运行全部测试套件确保未引入回归问题。若新增了功能，请同步补充对应的单元测试或集成测试用例，并更新相关文档中的描述。

3. 提交信息请遵循 Conventional Commits 规范，使用 `feat:`、`fix:`、`docs:`、`chore:` 等前缀，正文简要说明改动内容与原因。提交前请确保代码已通过 ESLint 检查。

4. 向原仓库发起 Pull Request，PR 描述中请清晰说明解决的问题或实现的功能，并关联相关 issue 编号（如有）。PR 需要至少一位维护者审核，审核通过后即合并。

5. 若您希望长期参与项目维护，可联系核心维护者申请成为协作者，后续可直接管理 issues 与合并经过充分审查的 PR。

## 常见问题

**问：Cinemeta Index 是否提供代理或翻墙功能以访问被屏蔽的站点？**  
答：不提供。本系统纯粹是目录导航，不包含任何网络代理、VPN、隧道或镜像转发功能。用户访问外部站点时直接通过本地网络环境进行，访问结果取决于用户所在网络条件与目标站点的可用状态。我们强烈建议用户遵守所在国家或地区的法律法规，合法使用互联网资源。

**问：如何更新收录的站点列表？我可以直接修改代码吗？**  
答：如果您是普通用户，可以在前端界面使用“导入 CSV”功能批量替换列表，或通过管理面板（若启用）逐个添加/删除条目。如果您是开发者或自部署用户，可以直接修改 `src/config/default.json` 中的预设列表，或通过环境变量 `SITES_JSON` 传入外部 JSON 数据源。所有改动不会影响其他用户的部署实例。

**问：可用性检查功能多久执行一次？是否会影响性能？**  
答：默认情况下，可用性检查脚本每 60 分钟触发一次，仅对收录站点发送轻量级 HEAD 请求，超时时间设为 5 秒。该过程在后台异步执行，不会阻塞正常页面访问。对于自托管用户，您可以通过环境变量 `CHECK_INTERVAL` 调整检查间隔，或通过 `CHECK_TIMEOUT` 调整超时阈值。我们不建议将间隔设为低于 10 分钟，以避免被目标站点误判为异常流量。
