# LinkVault

LinkVault 是一个面向技术内容创作者、开发者博客运营者以及开源项目维护者的外链资源管理与技术导航工具。它提供了一套标准化的外部链接整理、分类、版本追踪与健康状态监测方案，帮助用户解决多源技术链接分散、失效、难以维护的痛点。LinkVault 定位于轻量级技术资源汇总站点的快速构建，适用于个人知识库、团队技术文档门户以及社区驱动的学习资源导航项目。

## 功能概览

- **外链分类索引**：提供多级分类体系，支持按技术领域、资源类型、适用人群对链接进行精细化归类，便于快速定位目标资源。
- **链接健康监测**：定时探测已收录的外部链接可访问性，自动标记失效或响应超时的链接，生成健康报告，降低维护成本。
- **资源元数据提取**：自动抓取目标页面的标题、描述、关键词及最后更新时间，丰富链接索引信息，提升检索与筛选效率。
- **版本化快照记录**：对每个收录链接的页面内容进行周期性快照保存，记录变更历史，支持回溯特定时间点的资源状态。
- **多格式数据导出**：支持将链接索引导出为 JSON、YAML、Markdown 表格或 CSV 格式，便于迁移至其他文档系统或进行二次分析。
- **访问统计面板**：展示每个外链在站内的点击次数、最近访问时间及来源分布，辅助判断资源热度与内容价值。
- **标签与全文检索**：基于标题、描述、标签及分类进行全文关键词检索，配合标签筛选，提升大型导航站的资源查找效率。

## 应用场景

- **个人技术博客的推荐资源页**：技术博主可使用 LinkVault 快速生成一个动态更新的“友情链接”或“推荐工具”页面，自动监测链接有效性，避免读者访问死链。
- **开源项目文档的生态导航**：开源项目维护者可将项目依赖的文档、社区论坛、示例代码仓库等外部资源统一收录，并嵌入项目官网文档中，降低新贡献者的信息查找门槛。
- **技术社区的知识库门户**：技术社区运营团队可搭建基于 LinkVault 的社区资源中心，按话题（如前端、后端、运维、AI）分类整理优质外链，并提供搜索与推荐功能。
- **企业内部门户的技术手册汇总**：企业内部团队可将分散在多个外部 Wiki、代码仓库、设计规范站点的链接集中管理，形成统一的技术入口，减少信息孤岛。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，帮助您在本地快速启动 LinkVault 实例。

```bash
# 1. 克隆代码仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 2. 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 3. 初始化数据库并运行开发服务器
python manage.py migrate
python manage.py loaddata initial_links.json  # 加载示例链接数据
python manage.py runserver
```

访问 `http://127.0.0.1:8000` 即可查看 LinkVault 示例界面。默认管理员账号为 `admin`，密码 `linkvault2024`，首次登录后请及时修改。

| Python | 3.9 - 3.12 | 核心运行环境，推荐使用 3.11 以获取最佳性能 |

| Node.js | 18+ | 仅用于前端资产构建（CSS/JS 打包），运行时不需要 |
| Nginx | 1.24+ | 生产环境反向代理与静态文件服务，开发环境可忽略 |

## 常见问题

**Q：LinkVault 支持同时管理超过 10000 个外部链接吗？性能如何？**

A：支持。在 PostgreSQL 数据库配合 Redis 缓存的典型生产配置下，LinkVault 的链接检索与分类筛选响应时间在 200ms 以内（数据量 5 万条以内）。健康监测任务采用 Celery 分布式队列，可根据 Worker 数量水平扩展，建议每 6 小时执行一次全量扫描以平衡资源消耗。

**Q：我可以将 LinkVault 部署为纯静态站点吗？**

A：LinkVault 本身是一个 Django 动态应用，依赖后端 API 和数据库。但您可以使用其数据导出功能，将链接列表及元数据生成为 JSON 文件，再通过静态站点生成器（如 Hugo 或 VuePress）二次构建。我们提供了 `export_to_static.py` 脚本辅助此流程，详见文档 `/docs/admin/static-export.md`。

**Q：如何迁移已有的浏览器书签或 GitHub Star 列表？**

A：LinkVault 提供了 `scripts/migrate_legacy.py` 工具，支持导入 Chrome 书签导出的 HTML 文件、GitHub Star 的 JSON 列表以及普通 CSV 格式的链接表。运行 `python scripts/migrate_legacy.py --help` 查看具体用法。
